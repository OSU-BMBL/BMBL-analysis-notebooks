# AI Context: scRNAseq iPSC Branch (NOTCH1 Cardiac Differentiation)

## Quick Summary

This is a comprehensive multi-modal workflow for analyzing single-cell RNA-seq data from iPSC cardiac differentiation studies. It processes data from 12 samples (Control vs NOTCH1 KO) across 6 timepoints (Days 0, 2, 5, 10, 14, 30) to investigate cardiac development and the role of NOTCH1 signaling. The workflow includes standard scRNA-seq analysis, cell type annotation, differential expression, trajectory analysis (Monocle3), RNA velocity (velocyto), pathway enrichment, and gene regulatory network analysis.

**Analysis Type**: Single-cell RNA-seq iPSC differentiation (multi-timepoint, multi-condition)  
**Primary Tools**: Seurat, Monocle3, velocyto, CellChat, Harmony, MAST  
**Typical Runtime**: 2-4 hours for complete analysis  
**Memory Requirements**: 32GB minimum, 64GB recommended  
**Platform**: Local workstation (OSC compatible for most steps except velocyto)

## Data Flow

Data transformation pipeline from input to output:

```
10x Genomics output (12 samples, 6 timepoints x 2 conditions)
    ↓ (NOTCH1_iPSC_tutorial.rmd)
Quality control and filtering
    ↓
Integration with Harmony (batch correction)
    ↓
Dimensionality reduction (PCA → UMAP)
    ↓
Clustering and cell type annotation
    ↓
[Parallel analysis branches]
├── Differential expression (MAST)
├── Trajectory analysis (Monocle3)
├── RNA velocity analysis (Python/velocyto)
├── Cell-cell communication (CellChat)
└── Pathway enrichment (GSEA)
    ↓
Integrated results and visualizations
```

**Input Location**: `10x_counts/` directory with CellRanger outputs  
**Output Location**: Multiple directories - `rmarkdown/`, `jupyter/`, `velocity_loom/`  
**Intermediate Files**: 
- Seurat objects at each processing stage
- Loom files for velocity analysis
- Monocle3 CDS objects
- DEG tables and pathway results

## Common Modifications

### 1. Changing Timepoints or Conditions
**Task**: Analyze different timepoints or experimental conditions  
**File**: `jupyter/day*.ipynb` or `rmarkdown/` files  
**Line/Variable**: Sample filtering sections (varies by notebook)  
**Change**:
```python
# Current: Days 0-2 comparison
adata = adata[adata.obs['sample'].isin(['N1KO D0','N1KO D2','WT D0','WT D2'])]

# Modify for different days:
adata = adata[adata.obs['sample'].isin(['N1KO D5','N1KO D10','WT D5','WT D10'])]

# Or different conditions:
adata = adata[adata.obs['condition'].isin(['Treatment','Control'])]
```
**Impact**: Changes which samples are included in analysis. Must ensure metadata columns exist.  
**Testing**: Check `adata.obs['sample'].value_counts()` shows expected sample sizes.

### 2. Adjusting Quality Control Thresholds
**Task**: Modify cell filtering criteria  
**File**: `NOTCH1_iPSC_tutorial.rmd` (QC section)  
**Line/Variable**: Filter parameters (early in the Rmd)  
**Change**:
```r
# Default thresholds:
combined <- subset(combined, subset = nFeature_RNA > 200 & nFeature_RNA < 6000 & percent.mito < 20)

# More stringent:
combined <- subset(combined, subset = nFeature_RNA > 500 & nFeature_RNA < 5000 & percent.mito < 15)

# Less stringent (for lower quality data):
combined <- subset(combined, subset = nFeature_RNA > 100 & nFeature_RNA < 8000 & percent.mito < 25)
```
**Impact**: Changes number of cells retained. More stringent = higher quality but fewer cells.  
**Testing**: Check cell counts before/after filtering. Should retain >70% of cells typically.

### 3. Changing Integration Method
**Task**: Use different batch correction approach  
**File**: `NOTCH1_iPSC_tutorial.rmd` (integration section)  
**Line/Variable**: Harmony integration step  
**Change**:
```r
# Current: Harmony
combined <- combined %>% RunHarmony(group.by.vars = "orig.ident")

# Alternative: Seurat CCA
combined <- combined %>% 
  FindIntegrationAnchors(dims = 1:30) %>%
  IntegrateData()

# Alternative: No integration (if no batch effects)
# Skip integration, proceed directly to PCA
```
**Impact**: Changes how batch effects are handled. Harmony is recommended for this dataset.  
**Testing**: Check UMAP - cells should mix well across samples, not cluster by batch.

### 4. Modifying Clustering Resolution
**Task**: Change granularity of cell type clusters  
**File**: `NOTCH1_iPSC_tutorial.rmd` (clustering section)  
**Line/Variable**: `FindClusters()` resolution parameter  
**Change**:
```r
# Current:
combined <- FindClusters(combined, resolution = 0.8)

# Finer clusters (more cell types):
combined <- FindClusters(combined, resolution = 1.2)

# Coarser clusters (broader populations):
combined <- FindClusters(combined, resolution = 0.4)
```
**Impact**: Higher resolution = more clusters, potentially over-splitting. Lower = fewer clusters, potentially merging distinct types.  
**Testing**: Compare marker gene expression. Optimal resolution shows distinct marker patterns.

### 5. Customizing Cell Type Annotations
**Task**: Add or modify cell type definitions  
**File**: `2_cell_type_annotation.rmd`  
**Line/Variable**: Marker definitions and cluster assignments (around lines 100-150)  
**Change**:
```r
# Add new marker set:
cardiac_markers <- c("TNNT2", "MYH6", "MYH7", "TTN")

# Modify cluster assignments:
new_cluster_ids <- c(
  "iPSC", "Cardiac_mesoderm", "FHF_progenitor", 
  "Cardiomyocyte", "Cardiac_fibroblast"
)
names(new_cluster_ids) <- levels(combined)
combined <- RenameIdents(combined, new_cluster_ids)
```
**Impact**: Changes biological interpretation. Must be based on literature-supported markers.  
**Testing**: Verify marker gene expression matches assigned cell types in FeaturePlots.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **Memory requirements**: Full dataset (all 12 samples) requires 32-64GB RAM
  **Solution**: Subsample to specific timepoints for initial testing
  ```r
  combined <- subset(combined, subset = timepoint %in% c("D0", "D5", "D14"))
  ```

- **Velocyto OSC incompatibility**: RNA velocity requires Python velocyto which may not work on OSC
  **Solution**: Run velocity analysis locally, or use scVelo as alternative

- **Long runtime**: Complete workflow takes 2-4 hours
  **Solution**: Run in stages, save intermediate Seurat objects

### 🔶 Common Mistakes
- **Wrong sample metadata**: `sample_list.csv` must match directory names exactly
  **Why**: Case-sensitive, no extra spaces
  **Fix**: Verify CSV contents match `10x_counts/` directory names

- **Mixing spliced/unspliced**: Velocity analysis requires proper loom file format
  **Why**: Wrong file format = incorrect velocity estimates
  **Fix**: Use velocyto-generated loom files with spliced/unspliced layers

### 📋 Dependencies Between Steps
- Must complete QC before integration
- Must run integration before clustering
- Must annotate cell types before trajectory analysis
- Must have Seurat object before velocity analysis
- Must run velocity before CellRank analysis (in jupyter notebooks)

## Common Errors & Solutions

### Error: "cannot allocate vector of size X Gb"
**Cause**: Dataset too large for available memory  
**Solution**: 
```r
# Subsample for testing:
combined_small <- subset(combined, downsample = 500)

# Or analyze by timepoint:
for(tp in unique(combined$timepoint)) {
  tp_data <- subset(combined, subset = timepoint == tp)
  # Process separately
}
```
**Prevention**: Start with subset, gradually increase to full dataset

### Error: "Harmony convergence failed"
**Cause**: Strong batch effects or insufficient cells  
**Solution**: 
```r
# Increase theta for stronger correction:
combined <- RunHarmony(combined, group.by.vars = "orig.ident", theta = 3)

# Or try without harmony if batches already align:
combined <- RunPCA(combined)  # Skip harmony
```
**Prevention**: Check batch effect severity with `DimPlot(group.by = "orig.ident")` before integration

### Error: "No loom files found"
**Cause**: Velocity analysis not yet run  
**Solution**: Run velocyto first:
```bash
velocyto run10x -m repeat_msk.gtf sample_directory genes.gtf
```
**Prevention**: Follow velocyto installation and execution steps

### Error: "Color scheme not defined"
**Cause**: Color variables not loaded  
**Solution**: Define colors manually or load from functions file:
```r
source("../_common/functions.R")
# Or define manually:
sample_color <- c("WT D0" = "#0000FF", ...)
```
**Prevention**: Always run `source()` commands at start of scripts

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `nFeature_RNA min` | 200 | 100-500 | Minimum genes per cell |
| `nFeature_RNA max` | 6000 | 4000-8000 | Maximum genes per cell |
| `percent.mito` | 20 | 10-30 | Maximum mitochondrial % |
| `harmony.theta` | 2 | 1-5 | Integration strength |
| `cluster.resolution` | 0.8 | 0.4-1.5 | Clustering granularity |
| `dims` | 30 | 20-50 | PCA dimensions for analysis |

### When to Change Parameters
- **Increase QC thresholds**: For high-quality data, remove more low-quality cells
- **Decrease QC thresholds**: For difficult samples, retain more cells
- **Increase harmony.theta**: For strong batch effects
- **Increase resolution**: To identify rare subpopulations
- **Decrease resolution**: To merge over-split clusters
