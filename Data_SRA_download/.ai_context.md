## Quick Summary

Download raw sequencing data from SRA (Sequence Read Archive).

**Typical Runtime**: Varies by dataset size (hours to days)  
**Memory Required**: 8GB RAM  
**Input**: SRA accession numbers (e.g., SRR123456)  
**Output**: FASTQ files (.fastq.gz)

---

## Data Flow

```
SRA Accessions
    |
    v
prefetch (SRA toolkit)
    |
    v
SRA Files (.sra)
    |
    v
fastq-dump / fasterq-dump
    |
    v
FASTQ Files (.fastq.gz)
```

---

## Common Modifications

### 1. Parallel Downloads
**File**: `download_sra.sh` (line 25)

```bash
# Download multiple accessions in parallel
cat accessions.txt | parallel -j 4 prefetch {}
```

### 2. Split by Lane
**File**: `download_sra.sh` (line 40)

```bash
# Keep read groups (lanes) separate
fasterq-dump --split-files --include-technical SRR123456
```

---

## Gotchas & Warnings

1. **Disk Space**: SRA downloads require 3x the final FASTQ size (temp files)
2. **Aspera vs HTTPS**: Aspera is faster but requires installation
3. **Paired-end**: Use --split-files for paired-end data
4. **Check integrity**: Always verify MD5 checksums post-download
