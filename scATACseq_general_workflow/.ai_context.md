# AI Context: scATAC-seq General Workflow (Signac)

## Quick Summary

This workflow analyzes single-cell ATAC-seq (scATAC-seq) data using the Signac package. ATAC-seq measures chromatin accessibility - which regions of DNA are "open" and accessible to regulatory proteins. Think of it as mapping the "on/off switches" of genes across individual cells. The workflow includes quality control, peak calling, dimensionality reduction, and integration with scRNA-seq data for cell type annotation.

**Analysis Type**: Single-cell ATAC-seq (chromatin accessibility)  
**Primary Tools**: Signac, Seurat, EnsDb genome annotations  
**Typical Runtime**: 1-2 hours (depending on dataset size and fragment file processing)  
**Memory Requirements**: 16GB minimum, 32GB recommended (fragment files are large)  
**Platform**: Primarily OSC HPC (fragment files often >1GB)

## Data Flow

Data transformation pipeline from input to output:

```
10X Genomics scATAC-seq output
    ↓ (Signac_study.Rmd)
Peak/Cell matrix (HDF5 format)
    ↓
Fragment file (.tsv.gz) + index
    ↓
Create ChromatinAssay object
    ↓
QC metrics (TSS enrichment, nucleosome signal)
    ↓
Filtering cells by QC thresholds
    ↓
Normalization + Feature selection
    ↓
Dimensionality reduction (LSI - Latent Semantic Indexing)
    ↓
Clustering and UMAP visualization
    ↓
Gene activity scoring + annotation
    ↓
OPTIONAL: Integration with scRNA-seq reference
    ↓
Peak-level analysis and visualization
    ↓
Chromatin accessibility results + annotated cells
```

**Input Location**: `./Data/` folder (user must create and populate)  
**Output Location**: HTML report with embedded figures  
**Intermediate Files**: 
- ChromatinAssay object in Seurat format
- QC metadata (TSS enrichment, nucleosome signal per cell)

## Common Modifications

### 1. Changing Genome Build
**Task**: Switch between human (hg19/hg38) or mouse (mm10) genome annotations  
**File**: `Signac_study.Rmd`  
**Line/Variable**: Multiple locations - genome specification  
**Change**:
```r
# For hg38 instead of hg19:
# Line 107:
genome = 'hg38'

# Lines 56-60 - Install hg38 annotation:
if (!requireNamespace(c('BSgenome.Hsapiens.UCSC.hg38', 'EnsDb.Hsapiens.v86'), quietly = TRUE)){
  BiocManager::install(c('BSgenome.Hsapiens.UCSC.hg38', 'EnsDb.Hsapiens.v86'))
}

# Line 81:
library(EnsDb.Hsapiens.v86)  # Instead of v75

# Line 134:
annotations <- GetGRangesFromEnsDb(ensdb = EnsDb.Hsapiens.v86)

# Lines 137-138:
seqlevelsStyle(annotations) <- 'UCSC'
genome(annotations) <- "hg38"
```
**Impact**: Ensures peak annotation and gene activity scores use correct genomic coordinates. Mismatch = wrong gene assignments.  
**Testing**: Check `granges(pbmc)` output - chromosome names should match your genome (chr1 vs 1).

### 2. Adjusting QC Thresholds
**Task**: Change filtering cutoffs for TSS enrichment and nucleosome signal  
**File**: `Signac_study.Rmd`  
**Line/Variable**: QC filtering section (around line 180+)  
**Change**:
```r
# Default QC thresholds:
pbmc <- subset(
  x = pbmc,
  subset = nCount_peaks > 3000 &
    nCount_peaks < 20000 &
    nucleosome_signal < 2 &
    TSS.enrichment > 2
)

# Adjust based on your data distribution:
pbmc <- subset(
  x = pbmc,
  subset = nCount_peaks > 1000 &      # Lower minimum
    nCount_peaks < 30000 &             # Higher maximum
    nucleosome_signal < 2.5 &          # Slightly relaxed
    TSS.enrichment > 1.5               # Less stringent
)
```
**Impact**: Determines which cells pass quality control. Too strict = lose valid cells; too lenient = keep low-quality cells.  
**Testing**: Examine QC plots before filtering; check cell count after filtering (aim for 70-90% retention).

### 3. Using Custom Fragment File Paths
**Task**: Point to fragment files in a different location  
**File**: `Signac_study.Rmd`  
**Line/Variable**: `fragments` parameter (line 108)  
**Change**:
```r
# Default (tutorial data):
fragments = './Data/atac_v1_pbmc_10k_fragments.tsv.gz'

# Your data:
fragments = '/path/to/your/fragments.tsv.gz'
# Must also have .tbi index file in same location!
```
**Impact**: The fragment file is REQUIRED for many Signac functions (coverage plots, feature linkage). Without it, many analyses fail.  
**Testing**: `pbmc[['peaks']]` should show fragment file path; `Fragments(pbmc)` should return valid object.

**Task**: Call cell-type-specific peaks instead of using Cell Ranger peaks

## Gotchas & Warnings

### ⚠️ Critical Issues

- **Fragment file is mandatory**: Many Signac functions require the fragment file. If you only have the peak matrix (no fragments), you can't make coverage plots, calculate gene activities, or do footprinting.  
  **Solution**: Always keep fragments.tsv.gz + .tbi index. If files are huge, store on OSC and use symlinks.

- **Genome build mismatch**: Using hg19 data with hg38 annotations (or vice versa) places peaks on wrong genes.  
  **Solution**: Check Cell Ranger output for genome version; match annotations exactly.

- **Running out of disk space**: Fragment files are 1-5GB; multiple copies quickly fill disk.  
  **Solution**: Use symlinks to original Cell Ranger output; don't copy files.

### 🔶 Common Mistakes

- **Treating ATAC like RNA data**: ATAC data is sparser (more zeros), uses different normalization (TF-IDF, not log-normalization), and different dimensionality reduction (LSI, not PCA).  
  **Why**: Open chromatin regions are binary (open/closed), not continuous counts.  
  **Fix**: Follow Signac's workflow exactly; don't substitute Seurat RNA-seq defaults.

- **Using wrong blacklist**: The blacklist filters artifact regions. Using mouse blacklist on human data (or vice versa) removes valid peaks or keeps artifacts.  
  **Why**: Blacklists are genome-specific.  
  **Fix**: Use `Signac::blacklist_hg38`, `blacklist_hg19`, or `blacklist_mm10` as appropriate.

- **Not checking nucleosome signal**: High nucleosome signal (>4) often indicates dead cells or debris.  
  **Why**: Living cells have clear nucleosome phasing; dead cells have random fragmentation.  
  **Fix**: Always plot nucleosome signal distribution before filtering.

### 📋 Dependencies Between Steps

- Must load genome annotations (Step 1) before gene activity analysis (Step 6)
- QC metrics must be calculated (Step 2) before QC filtering
- Dimensionality reduction (Step 3) must precede clustering (Step 4)
- scRNA-seq reference must be processed before integration (Step 6)

## Common Errors & Solutions

### Error: "cannot open file 'fragments.tsv.gz'"
**Cause**: Fragment file path incorrect or file doesn't exist  
  **Solution**: Verify path; ensure both .tsv.gz and .tbi index exist.  
  **Prevention**: Use absolute paths or check `file.exists()` before loading.

### Error: "invalid class 'GRanges' object"
**Cause**: Genome build mismatch between data and annotations  
  **Solution**: Check `genome(pbmc)` and `genome(annotations)` match.  
  **Prevention**: Always set genome consistently: `genome = 'hg38'` and `genome(annotations) <- "hg38"`.

### Error: "no method for coercing this S4 class to a vector"
**Cause**: Trying to use Seurat RNA-seq functions on ChromatinAssay  
  **Solution**: Use Signac-specific functions: `GeneActivity()` not `GetAssayData()`.  
  **Prevention**: Check assay type: `class(pbmc[['peaks']])` should be "ChromatinAssay".

### Error: "Error in .getOneSeqFromBSgenome"
**Cause**: BSgenome package for specified genome not installed  
  **Solution**: Install correct genome package:
  ```r
  BiocManager::install("BSgenome.Hsapiens.UCSC.hg38")
  ```
  **Prevention**: Install all genome packages mentioned in "Pre-requisite packages installation" section.

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `min.cells` | 10 | 5-50 | Minimum cells expressing a peak to keep it |
| `min.features` | 200 | 100-500 | Minimum peaks per cell to keep cell |
| `dim` (LSI) | 2:30 | 2:50 | LSI components for UMAP. Skip first (technical) |
| `resolution` | 0.8 | 0.4-1.2 | Clustering resolution |

### When to Change Parameters
- **Increase `min.cells`**: When too many rare peaks clutter analysis
- **Decrease `min.features`**: When cell quality is lower (degraded samples)
- **Use dims 2:50**: For large datasets with diverse cell types
- **Use dims 2:15**: For homogeneous populations
