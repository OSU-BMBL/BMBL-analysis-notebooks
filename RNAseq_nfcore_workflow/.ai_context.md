# AI Context: RNA-seq nf-core/rnaseq Workflow

## Quick Summary

This workflow processes bulk RNA sequencing data using the nf-core/rnaseq pipeline - a standardized, community-curated Nextflow pipeline. Unlike other BMBL workflows that run R scripts locally, this workflow executes a pre-built pipeline that handles everything from quality control to alignment to quantification. You provide FASTQ files and a samplesheet; the pipeline produces aligned BAMs, gene count matrices, and comprehensive QC reports. Think of it as a pipeline-as-a-service - the nf-core team maintains the software, you just provide data.

**Analysis Type**: Bulk RNA-seq (preprocessing and quantification)  
**Primary Tools**: Nextflow, nf-core/rnaseq, Singularity containers  
**Typical Runtime**: 4-8 hours for a typical experiment (6 samples, 20M reads each)  
**Memory Requirements**: OSC cluster recommended (80GB+ for STAR alignment)  
**Platform**: OSC HPC (Nextflow with Singularity)

## Data Flow

Data transformation pipeline from input to output:

```
Raw FASTQ files (.fastq.gz)
    ↓ (nextflow run nf-core/rnaseq)
Samplesheet.csv (defines sample metadata)
    ↓
Quality Control (FastQC)
    ↓
Adapter Trimming (Trim Galore!)
    ↓
Alignment (STAR to reference genome)
    ↓
Gene Quantification (Salmon + featureCounts)
    ↓
QC Aggregation (MultiQC report)
    ↓
STAR_salmon/ output directory
    ├── merged_gene_counts/   ← Count matrix for DESeq2
    ├── deseq2_qc/            ← QC plots
    ├── multiqc/              ← Comprehensive QC report
    └── all_sorted_BAM/       ← Aligned reads
```

**Input Location**: User-provided FASTQ files + samplesheet.csv  
**Output Location**: `outdir/` directory (user-specified)  
**Intermediate Files**: Processed in work/ directory (auto-cleaned)

## Common Modifications

### 1. Creating the Samplesheet
**Task**: Define samples and their FASTQ file paths for the pipeline  
**File**: `samplesheet.csv` (you create this)  
**Format**:
```csv
sample,fastq_1,fastq_2,strandedness
CONTROL_REP1,/path/to/ctrl1_R1.fastq.gz,/path/to/ctrl1_R2.fastq.gz,forward
CONTROL_REP2,/path/to/ctrl2_R1.fastq.gz,/path/to/ctrl2_R2.fastq.gz,forward
TREATMENT_REP1,/path/to/treat1_R1.fastq.gz,/path/to/treat1_R2.fastq.gz,reverse
```
**Change**: Add all your samples with correct paths  
**Impact**: The pipeline processes every sample in the samplesheet. Missing samples = incomplete analysis.  
**Testing**: Run `cat samplesheet.csv` to verify format; check file paths exist with `ls`.

### 2. Choosing the Genome
**Task**: Select reference genome for alignment  
**File**: Command line parameter  
**Change**:
```bash
# For human:
--genome GRCh38

# For mouse:
--genome GRCm38

# For other genomes, use iGenomes:
--genome GRCh37  # Human hg19
--genome GRCm39  # Mouse mm39
```
**Impact**: Determines which reference genome STAR aligns to. Wrong genome = wrong alignments.  
**Testing**: Check MultiQC report - alignment rate should be >70% for good quality data.

### 3. Specifying Strandedness
**Task**: Tell pipeline if your library prep is stranded  
**File**: `samplesheet.csv` strandedness column  
**Change**:
```csv
# Options:
forward    # Standard Illumina stranded (dUTP method)
reverse    # Reverse stranded
auto       # Let pipeline detect (slower but safe)
unstranded # Non-stranded library
```
**Impact**: Affects Salmon quantification accuracy. Wrong setting = incorrect counts.  
**Testing**: Use `auto` first time; pipeline will report detected strandedness. Use that for future runs.

### 4. Custom Output Directory
**Task**: Change where results are saved  
**File**: Command line parameter  
**Change**:
```bash
# Default:
--outdir ./results

# Custom:
--outdir /fs/scratch/PCON0022/yourname/rnaseq_results
```
**Impact**: Results location. Must have write permissions and sufficient disk space (~10GB per sample).  
**Testing**: Run `mkdir -p /your/outdir/path` first to verify permissions.

### 5. Resuming Failed Runs
**Task**: Continue a failed run without starting over  
**File**: Command line parameter  
**Change**:
```bash
# Add -resume flag:
nextflow run nf-core/rnaseq \
   --input samplesheet.csv \
   --outdir results \
   --genome GRCh38 \
   -profile singularity \
   -resume
```
**Impact**: Saves time and compute by reusing completed steps.  
**Testing**: Check console output - should show "Cached" for completed processes.

## Gotchas & Warnings

### ⚠️ Critical Issues

- **Do NOT run on login nodes**: Nextflow parallelizes across many CPUs. Running on OSC login nodes violates policy and will be killed.  
  **Solution**: Always submit as a batch job (see example SLURM script in README.md).

- **Disk space requirements**: Each sample generates ~10GB of output (BAMs, counts, QC). A 20-sample experiment needs 200GB+.  
  **Solution**: Use `/fs/scratch/` or `/fs/ess/` which have larger quotas than home directories.

- **Samplesheet MUST be CSV**: Excel files or TSV files will cause cryptic errors.  
  **Solution**: Use plain text CSV format. Check with `file samplesheet.csv` - should show "ASCII text".

### 🔶 Common Mistakes

- **Absolute vs relative paths in samplesheet**: Relative paths (./data/file.fastq.gz) fail because pipeline changes working directory.  
  **Why**: Nextflow runs in its own work directory.  
  **Fix**: Always use absolute paths: `/fs/scratch/PCON0022/username/data/file.fastq.gz`

- **Forgetting strandedness**: Default is unstranded, but most modern protocols are stranded.  
  **Why**: Unstranded on stranded data works but loses power; stranded on unstranded data gives wrong results.  
  **Fix**: Check your library prep kit documentation; use `auto` if unsure.

- **Not checking MultiQC before downstream analysis**: MultiQC aggregates all QC metrics. Issues here propagate to DESeq2.  
  **Why**: Poor quality data in = poor results out.  
  **Fix**: Always review multiqc_report.html before proceeding to differential expression.

### 📋 Dependencies Between Steps

- Pipeline handles all dependencies internally
- No local steps to coordinate
- Just: prepare samplesheet → run pipeline → review results → proceed to DESeq2

Validate inputs without processing:
```bash
nextflow run nf-core/rnaseq \
   --input samplesheet.csv \
   --outdir results \
   --genome GRCh38 \
   -profile singularity \
   --validate_params
```

**Expected**: Pipeline validates samplesheet format and exits without error.

### Full Test
Run with full samplesheet

- [ ] `multiqc/multiqc_report.html` - Comprehensive QC
- [ ] `star_salmon/merged_gene_counts/counts.tsv` - Count matrix

## Common Errors & Solutions

### Error: "Cannot find file: samplesheet.csv"
**Cause**: Wrong path to samplesheet  
**Solution**: Use absolute path: `--input /absolute/path/to/samplesheet.csv`  
**Prevention**: Always use `pwd` to get current directory, then full path.

### Error: "No such file: file.fastq.gz"
**Cause**: Relative path in samplesheet or file doesn't exist  
**Solution**: Use `ls` to verify files exist; use absolute paths in samplesheet.  
**Prevention**: Test paths with `ls $(cat samplesheet.csv | cut -d',' -f2 | tail -n +2)`.

### Error: "Process exceeded running time limit"
**Cause**: SLURM time limit too short  
**Solution**: Increase `#SBATCH --time` in your submit script (try 40:00:00).  
**Prevention**: Estimate 30-60 minutes per sample; add buffer.

### Error: "Insufficient memory"
**Cause**: STAR alignment needs more RAM than requested  
**Solution**: Increase `#SBATCH --mem` to 80GB or more.  
**Prevention**: Human genome alignment typically needs 40-50GB for STAR.

## Parameter Guide

### Key Parameters
| Parameter | Default | Options | Description |
|-----------|---------|---------|-------------|
| `--genome` | none | GRCh38, GRCm38, etc. | Reference genome |
| `--aligner` | star_salmon | star_rsem, hisat2 | Alignment method |
| `--pseudo_aligner` | salmon | none, kallisto | Pseudo-alignment |
| `--skip_qc` | false | true | Skip FastQC (faster) |

### When to Change Parameters
- **Use `--aligner star_rsem`**: If you need isoform-level counts
- **Use `--pseudo_aligner none`**: If you only want STAR counts (faster)
- **Use `--skip_qc`**: If you already ran FastQC separately
