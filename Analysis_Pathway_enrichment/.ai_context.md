# AI Context: Pathway Enrichment Analysis

## Quick Summary

This workflow performs functional enrichment analysis on gene sets using clusterProfiler. It identifies over-represented biological pathways and Gene Ontology (GO) terms in a list of genes (typically differentially expressed genes). The workflow supports both GO enrichment (Biological Process, Molecular Function, Cellular Component) and Reactome pathway analysis, with comprehensive visualization options including bar plots, dot plots, gene-concept networks, and enrichment maps.

**Analysis Type**: Functional enrichment analysis (post-DEG analysis)  
**Primary Tools**: clusterProfiler, ReactomePA, org.Hs.eg.db, enrichplot  
**Typical Runtime**: 5-15 minutes  
**Memory Requirements**: 8GB minimum, 16GB recommended for large gene lists  
**Platform**: Local or OSC HPC (R/RStudio)

---

## Data Flow

Data transformation pipeline from input to output:

```
Gene list (typically DEGs from prior analysis)
    ↓ (pathway_enrichment.Rmd)
Convert gene symbols to Entrez IDs
    ↓
Perform GO over-representation test
    ↓
Perform Reactome pathway enrichment
    ↓
Adjust p-values (Benjamini-Hochberg)
    ↓
Generate visualizations (bar, dot, network plots)
    ↓
HTML report with enrichment results
```

**Input Location**: Gene list in R (vector or from file)  
**Output Location**: `pathway_enrichment.html` + result objects  
**Intermediate Files**: None (all in R environment)

## Common Modifications

### 1. Using Your Own Gene List
**Task**: Analyze custom gene list instead of example  
**File**: `pathway_enrichment.Rmd`  
**Line/Variable**: Gene loading section (around lines 39-53)  
**Change**:
```r
# Current: Example gene list
example_gene_list <- c("GPX3", "GLRX", "LBP", ...)

# From DEG results file:
deg_results <- read.csv("your_degs.csv")
gene_list <- deg_results$gene_symbol[deg_results$padj < 0.05]

# From file:
gene_list <- readLines("your_genes.txt")

# From Seurat object markers:
markers <- FindAllMarkers(seurat_obj, only.pos = TRUE)
gene_list <- markers$gene[markers$cluster == "Cluster1"]
```
**Impact**: Input gene list determines enrichment results.  
**Testing**: Check `length(gene_list)` - typically 50-1000 genes works best.

### 2. Changing Significance Thresholds
**Task**: Adjust p-value and q-value cutoffs  
**File**: `pathway_enrichment.Rmd`  
**Line/Variable**: `enrichGO()` and `enrichPathway()` calls (lines 74-82, 94)  
**Change**:
```r
# Default (stringent):
enrichGO(..., pvalueCutoff = 0.05, qvalueCutoff = 0.05)
enrichPathway(..., pvalueCutoff = 0.05)

# More stringent (publication quality):
enrichGO(..., pvalueCutoff = 0.01, qvalueCutoff = 0.01)
enrichPathway(..., pvalueCutoff = 0.01)

# Less stringent (exploratory):
enrichGO(..., pvalueCutoff = 0.10, qvalueCutoff = 0.10)
enrichPathway(..., pvalueCutoff = 0.10)
```
**Impact**: Stringent thresholds = fewer, more significant pathways. Lenient = more pathways but may include false positives.  
**Testing**: Check number of significant terms - aim for 5-50 terms.

### 3. Selecting GO Ontology Category
**Task**: Focus on specific GO aspect (BP, MF, or CC)  
**File**: `pathway_enrichment.Rmd`  
**Line/Variable**: `enrichGO()` ont parameter (line 77)  
**Change**:
```r
# Current: Biological Process
gobp_result <- enrichGO(..., ont = "BP")

# Molecular Function:
gomf_result <- enrichGO(..., ont = "MF")

# Cellular Component:
gocc_result <- enrichGO(..., ont = "CC")

# Run all three:
gobp_result <- enrichGO(..., ont = "BP")
gomf_result <- enrichGO(..., ont = "MF")
gocc_result <- enrichGO(..., ont = "CC")
```
**Impact**: BP = what genes do; MF = how they do it; CC = where they act.  
**Testing**: BP usually most informative; MF good for enzyme analysis; CC for localization.

### 4. Using Different Organism Database
**Task**: Analyze non-human gene lists  
**File**: `pathway_enrichment.Rmd`  
**Line/Variable**: `bitr()` and `enrichGO()` OrgDb parameter (lines 47, 76)  
**Change**:
```r
# Current: Human
library(org.Hs.eg.db)
example_gene_mapping = bitr(..., OrgDb = "org.Hs.eg.db")
gobp_result <- enrichGO(..., OrgDb = org.Hs.eg.db)

# Mouse:
library(org.Mm.eg.db)
gene_mapping = bitr(..., OrgDb = "org.Mm.eg.db")
gobp_result <- enrichGO(..., OrgDb = org.Mm.eg.db)

# Rat:
library(org.Rn.eg.db)
gobp_result <- enrichGO(..., OrgDb = org.Rn.eg.db)

# Find available:
library(AnnotationHub)
ah <- AnnotationHub()
query(ah, "OrgDb")
```
**Impact**: Must match organism of your gene list.  
**Testing**: Check that gene ID conversion works (`bitr()` output).

### 5. Changing Background Gene Set
**Task**: Use custom background instead of all genes  
**File**: `pathway_enrichment.Rmd`  
**Line/Variable**: `enrichGO()` universe parameter (line 75)  
**Change**:
```r
# Current: Uses all genes in org.Hs.eg.db
gobp_result <- enrichGO(gene = example_gene_id,
                universe = names(geneList),  # All genes
                ...)

# Custom background (e.g., all expressed genes):
all_expressed_genes <- read.csv("expressed_genes.csv")$gene
universe_ids <- bitr(all_expressed_genes, fromType = "SYMBOL", 
                     toType = "ENTREZID", OrgDb = "org.Hs.eg.db")$ENTREZID

gobp_result <- enrichGO(gene = example_gene_id,
                universe = universe_ids,
                ...)
```
**Impact**: More accurate enrichment when background is all expressed genes in your experiment (not all genome genes).  
**Testing**: Results should change slightly - more specific with custom background.

### 6. Customizing Visualizations
**Task**: Modify plot appearance for publication  
**File**: `pathway_enrichment.Rmd`  
**Line/Variable**: Plotting sections (lines 102-138)  
**Change**:
```r
# Default dot plot:
dotplot(reactome_result, showCategory = 30)

# Publication-ready:
dotplot(reactome_result, 
        showCategory = 15,           # Fewer categories
        font.size = 12,              # Larger text
        title = "Reactome Pathways") + 
  theme_bw() +                     # Clean theme
  theme(axis.text.y = element_text(size = 10))

# Save high-res:
ggsave("enrichment_plot.pdf", width = 8, height = 10, dpi = 300)
```
**Impact**: Creates publication-quality figures.  
**Testing**: Check text is readable, labels aren't cut off.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **Gene ID conversion failures**: Not all symbols convert to Entrez IDs
  **Solution**: Check conversion rate:
  ```r
  print(paste("Converted:", nrow(gene_mapping), "of", length(gene_list)))
  ```
  **Fix**: Remove NA mappings before enrichment

- **Universe set too small**: Background must include most genome genes
  **Solution**: Use all expressed genes or full organism database
  **Why**: Small universe = inflated enrichment scores

### 🔶 Common Mistakes
- **Using raw p-values instead of adjusted**: Always use padj/qvalue
  **Why**: Multiple testing correction essential
  **Fix**: `pAdjustMethod = "BH"` (Benjamini-Hochberg) is default - don't change to "none"

- **Too few input genes**: < 10 genes = unreliable enrichment
  **Why**: Statistical power too low
  **Fix**: Use larger gene sets (> 50 genes)

### 📋 Dependencies Between Steps
- Must convert gene symbols to Entrez IDs before enrichment
- Must have `org.Hs.eg.db` (or other organism) installed
- Gene list must be character vector

## Common Errors & Solutions

### Error: "No gene can be mapped"
**Cause**: Gene symbols don't match database  
**Solution**: 
```r
# Check valid symbols:
library(org.Hs.eg.db)
keytypes(org.Hs.eg.db)  # See available ID types

# May need to convert from different format:
gene_list <- bitr(gene_list, fromType = "ENSEMBL", 
                  toType = "ENTREZID", OrgDb = "org.Hs.eg.db")
```
**Prevention**: Check ID type before conversion

### Error: "No enrichment found"
**Cause**: No significant pathways at given thresholds  
**Solution**: 
```r
# Relax thresholds:
gobp_result <- enrichGO(..., pvalueCutoff = 0.10, qvalueCutoff = 0.10)
```
**Prevention**: Ensure input gene list is biologically coherent

### Error: "OrgDb not found"
**Cause**: Organism database not installed  
**Solution**: 
```r
BiocManager::install("org.Hs.eg.db")
```
**Prevention**: Run `0_install_packages.R` first

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `pvalueCutoff` | 0.05 | 0.01-0.20 | Raw p-value threshold |
| `qvalueCutoff` | 0.05 | 0.01-0.20 | Adjusted p-value threshold |
| `pAdjustMethod` | "BH" | "BH"/"bonferroni"/"fdr" | Multiple testing correction |
| `ont` | "BP" | "BP"/"MF"/"CC"/"ALL" | GO ontology |
| `minGSSize` | 10 | 1-100 | Minimum genes in pathway |
| `maxGSSize` | 500 | 100-1000 | Maximum genes in pathway |

### When to Change Parameters
- **Increase cutoffs**: No significant results found
- **Decrease cutoffs**: Too many results, want only strongest
- **Use bonferroni**: Very strict control (conservative)
- **Adjust size limits**: Remove very small/large pathways
