# AI Context: Data GEO Download

## Quick Summary

This utility workflow downloads sample-level count matrices from NCBI GEO (Gene Expression Omnibus) using HTTPS instead of FTP. It's designed for downloading scRNA-seq data when per-sample files are needed rather than combined archives. The tool handles rate limiting, resume capabilities, and provides progress reporting for batch downloads of GEO datasets.

**Analysis Type**: Data acquisition utility  
**Primary Tools**: Python requests library, HTTPS  
**Typical Runtime**: Varies by dataset size (minutes to hours)  
**Memory Requirements**: Minimal (< 1GB)  
**Platform**: Any system with Python (local, OSC, cloud)

---

## Data Flow

Data transformation pipeline from input to output:

```
GEO Series ID or sample list (JSON)
    ↓ (1_download_samples.py)
Parse GSM IDs and filenames
    ↓
Build HTTPS download URLs
    ↓
Download each sample with rate limiting
    ↓
Save to output directory
    ↓
Verify downloads / resume if interrupted
```

**Input Location**: Command line arguments or JSON file  
**Output Location**: User-specified directory  
**Intermediate Files**: None (direct download to destination)

## Common Modifications

### 1. Creating Sample List JSON
**Task**: Define which GSM samples to download  
**File**: `samples.json` (user-created)  
**Format**:
```json
{
  "GSM4716780": "GSM4716780_sample1_counts.tsv.gz",
  "GSM4716781": "GSM4716781_sample2_counts.tsv.gz",
  "GSM4716782": "GSM4716782_sample3_counts.tsv.gz"
}
```
**How to create**:
1. Find your GEO series (e.g., GSE158699)
2. Click on individual GSM samples
3. Note the "Supplementary file" names
4. Create JSON mapping GSM ID to filename

**Change**: Add/remove samples as needed  
**Impact**: Determines which samples are downloaded  
**Testing**: Validate JSON syntax:
```bash
python -m json.tool samples.json
```

### 2. Adjusting Download Delay
**Task**: Control rate limiting to avoid GEO blocks  
**File**: `1_download_samples.py`  
**Line/Variable**: `--delay` parameter (command line)  
**Change**:
```bash
# Default 0.5 seconds between downloads:
python 1_download_samples.py --samples samples.json --output ./data/

# Slower (for large batches):
python 1_download_samples.py --samples samples.json --output ./data/ --delay 2.0

# Faster (for small batches):
python 1_download_samples.py --samples samples.json --output ./data/ --delay 0.1
```
**Impact**: Longer delays = slower but more reliable. Too fast = IP may be blocked.  
**Testing**: Start with 0.5s, increase if getting connection errors.

### 3. Resuming Interrupted Downloads
**Task**: Continue download after interruption  
**File**: `1_download_samples.py`  
**Behavior**: Automatic - script skips existing files  
**Usage**:
```bash
# If download stopped, just re-run:
python 1_download_samples.py --samples samples.json --output ./data/

# Will show:
# [3/10] Skipping GSM4716782 (exists)
```
**Impact**: Saves time by not re-downloading completed files.  
**Testing**: Check file sizes match expected sizes from GEO.

### 4. Changing Timeout
**Task**: Adjust timeout for slow connections  
**File**: `1_download_samples.py`  
**Line/Variable**: `download_file()` function, `timeout` parameter  
**Change**:
```python
# Current (120 seconds):
def download_file(url: str, filepath: Path, timeout: int = 120) -> bool:

# For slow connections:
def download_file(url: str, filepath: Path, timeout: int = 300) -> bool:
```
**Impact**: Prevents premature timeout on slow connections.  
**Testing**: If downloads fail with timeout errors, increase this value.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **Filename must match exactly**: GEO is case-sensitive
  **Solution**: Copy-paste filenames from GEO website
  **Verification**: Compare `samples.json` values with GEO supplementary files list

- **GEO rate limiting**: Too many requests = temporary IP block
  **Solution**: Use `--delay` parameter, start with 0.5-1.0 seconds
  **Recovery**: Wait 10-15 minutes, try again with longer delay

### 🔶 Common Mistakes
- **Wrong file type**: Trying to download FASTQ files with this tool
  **Why**: This tool is for processed count matrices, not raw reads
  **Fix**: For FASTQ, use SRA toolkit (`prefetch`, `fasterq-dump`)

- **Not checking file sizes**: Downloads may complete but be corrupted
  **Why**: Network interruptions can cause incomplete files
  **Fix**: Compare downloaded file sizes with GEO listing
  ```bash
  ls -lh ./data/*.gz
  ```

### 📋 Dependencies Between Steps
- Must have valid GSM IDs before downloading
- Must create JSON file before running script
- Output directory is created automatically if it doesn't exist

## Common Errors & Solutions

### Error: "Connection refused" or timeout
**Cause**: GEO servers busy or rate limiting  
**Solution**: 
```bash
# Increase delay:
python 1_download_samples.py --samples samples.json --output ./data/ --delay 2.0
```
**Prevention**: Use delays > 0.5s for batches > 10 samples

### Error: "404 Not Found"
**Cause**: Filename mismatch or GSM doesn't exist  
**Solution**: 
- Verify GSM ID on GEO website
- Check exact filename including version suffixes (e.g., `_v2`)
**Prevention**: Copy-paste from GEO supplementary files table

### Error: "JSON decode error"
**Cause**: Invalid JSON syntax in samples file  
**Solution**: Validate JSON:
```bash
python -m json.tool samples.json
```
**Prevention**: Use proper JSON format with quotes and commas

### Error: "Permission denied"
**Cause**: Can't write to output directory  
**Solution**: Check directory permissions:
```bash
mkdir -p ./data
chmod 755 ./data
```

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `--delay` | 0.5 | 0.1-5.0 | Seconds between downloads |
| `--timeout` | 120 | 60-600 | Download timeout (seconds) |
| `--output` | Required | Path | Output directory |
| `--samples` | Required | JSON file | Sample mapping file |

### When to Change Parameters
- **Increase delay**: For large batches (> 50 samples) or if getting blocked
- **Increase timeout**: For slow connections or large files
- **Use smaller batches**: If experiencing frequent failures
