# AI Context: scRNAseq Immune Branch

## Quick Summary

This workflow provides immune-specific cell type annotation for single-cell RNA-seq data using curated marker genes. It's a specialized branch of the general scRNA-seq workflow that focuses on immune and blood cell populations. The workflow loads a pre-processed Seurat object, visualizes known immune marker gene expression, and assigns biological cell types to clusters based on established immune cell markers.

**Analysis Type**: Single-cell RNA-seq immune cell annotation  
**Primary Tools**: Seurat, dittoSeq, qs (quick serialization)  
**Typical Runtime**: 5-15 minutes  
**Memory Requirements**: 8GB minimum, 16GB recommended  
**Platform**: Both OSC HPC and Local (RStudio)

---

## Data Flow

Data transformation pipeline from input to output:

```
Preprocessed Seurat object (combined.qsave)
    ↓ (2_annotate_cell_type.rmd)
Load immune markers reference table
    ↓
Visualize marker expression across clusters
    ↓
Manual cluster annotation based on markers
    ↓
Assign cell types to clusters
    ↓
Generate UMAPs with cell type labels
    ↓
Create cell type proportion bar plots
    ↓
Save annotated Seurat object (combined.qsave)
```

**Input Location**: `../scRNAseq_preprocess_main/combined.qsave`  
**Output Location**: `./combined.qsave` (updated with cell_type metadata)  
**Intermediate Files**: None

## Common Modifications

### 1. Changing Input File Path
**Task**: Use a different input Seurat object  
**File**: `2_annotate_cell_type.rmd`  
**Line/Variable**: Data loading section (around line 44)  
**Change**:
```r
# Default path:
combined <- qs::qread(paste0("../scRNAseq_preprocess_main/combined.qsave"))

# Change to your path:
combined <- qs::qread("path/to/your/seurat_object.qsave")
# or for .rds format:
combined <- readRDS("path/to/your/seurat_object.rds")
```
**Impact**: Must point to a valid Seurat object with clustering already performed.  
**Testing**: Check `combined` object loads successfully and has `seurat_clusters` column.

### 2. Adding Custom Immune Markers
**Task**: Include additional marker genes for annotation  
**File**: `2_annotate_cell_type.rmd`  
**Line/Variable**: Marker visualization sections (multiple FeaturePlot calls, around lines 97-209)  
**Change**:
```r
# Add new marker visualization after existing ones:
FeaturePlot(combined, 
            reduction = "umap", 
            features = c("YOUR_MARKER1", "YOUR_MARKER2"), 
            order = TRUE,
            min.cutoff = 'q10', 
            label = TRUE)
```
**Also update the annotation section (around line 218-226)**:
```r
# Add new cell type assignment:
tmp_levels[c(X)] <- "Your New Cell Type"  # X = cluster number(s)
```
**Impact**: Extends annotation capability to additional immune populations.  
**Testing**: Verify new markers show expected expression patterns in FeaturePlots.

### 3. Modifying Cell Type Assignments
**Task**: Change how clusters map to cell types  
**File**: `2_annotate_cell_type.rmd`  
**Line/Variable**: Annotation section (around lines 216-226)  
**Change**:
```r
# Current assignments:
tmp_levels[c(1)] <- "CD14+ monocytes"
tmp_levels[c(7)] <- "FCGR3A+ monocytes"

# Modify to your assignments:
tmp_levels[c(1,2)] <- "CD14+ monocytes"  # Combine clusters 1 and 2
tmp_levels[c(7)] <- "Non-classical monocytes"  # Rename
# Add new assignment:
tmp_levels[c(11)] <- "Neutrophils"
```
**Impact**: Changes final cell type labels. Critical to match marker expression patterns.  
**Testing**: Verify UMAP plot shows sensible cell type distributions matching biology.

### 4. Changing Color Schemes
**Task**: Use different colors for cell types or samples  
**File**: `2_annotate_cell_type.rmd`  
**Line/Variable**: DimPlot calls (around lines 56, 240, 253)  
**Change**:
```r
# Define custom colors before plotting:
cell_type_color <- c(
  "CD14+ monocytes" = "#1f77b4",
  "FCGR3A+ monocytes" = "#ff7f0e",
  "B cells" = "#2ca02c",
  # ... add all cell types
)

# Use in plots:
DimPlot(combined, cols = cell_type_color)
```
**Impact**: Changes visualization aesthetics. Important for publication figures.  
**Testing**: Check plots render with expected colors.

### 5. Adjusting Expression Cutoffs
**Task**: Change minimum expression thresholds for marker visualization  
**File**: `2_annotate_cell_type.rmd`  
**Line/Variable**: FeaturePlot calls (around lines 97-209)  
**Change**:
```r
# Default:
FeaturePlot(combined, features = c("CD14", "LYZ"), 
            min.cutoff = 'q10', ...)  # 10th percentile

# More sensitive (show more cells):
FeaturePlot(combined, features = c("CD14", "LYZ"), 
            min.cutoff = 'q1', ...)   # 1st percentile

# Less sensitive (show only high expressers):
FeaturePlot(combined, features = c("CD14", "LYZ"), 
            min.cutoff = 'q25', ...)  # 25th percentile
```
**Impact**: Changes which cells are colored in marker expression plots.  
**Testing**: Ensure markers are visible but background noise is minimized.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **Cluster indexing**: Seurat clusters start at 0, but R indexing starts at 1
  **Solution**: The code handles this (note comments on line 214), but be careful when modifying
  ```r
  # Line 214 comment: "Note: the index starts at 1, but the seurat clusters starts at 0"
  tmp_levels[c(1)] <- "CD14+ monocytes"  # This assigns cluster 0!
  ```

- **Input object must have clustering**: This workflow requires pre-clustered data
  **Solution**: Run preprocessing workflow first (scRNAseq_general_workflow step 1-2)

### 🔶 Common Mistakes
- **Forgetting to switch default assay**: Visualization requires RNA assay, not integrated
  **Why**: Integrated assay doesn't have raw expression values
  **Fix**: Code already includes `DefaultAssay(combined) <- "RNA"` - don't remove it!

- **Inconsistent marker combinations**: Assigning cell types without checking all markers
  **Why**: Single markers can be ambiguous (e.g., CD3D in both CD4+ and CD8+ T cells)
  **Fix**: Use combination of markers as shown in the marker table (lines 68-80)

### 📋 Dependencies Between Steps
- Input file must exist at specified path BEFORE running
- Must load `_common/functions.R` for color schemes
- Must normalize RNA data before visualization (line 88)
- Cell type assignments must match cluster indices exactly

## Common Errors & Solutions

### Error: "cannot open file '../scRNAseq_preprocess_main/combined.qsave'"
**Cause**: Input file doesn't exist at expected path  
**Solution**: 
```r
# Check file location:
file.exists("../scRNAseq_preprocess_main/combined.qsave")

# If false, update path in line 44:
combined <- qs::qread("correct/path/to/your/file.qsave")
```
**Prevention**: Run preprocessing workflow first, or verify input file location

### Error: "could not find function 'dittoBarPlot'"
**Cause**: dittoSeq package not installed  
**Solution**: `install.packages("dittoSeq")`  
**Prevention**: Install all packages listed in README

### Error: "object 'cell_type_color' not found"
**Cause**: `_common/functions.R` not loaded or doesn't define colors  
**Solution**: 
```r
# Either ensure functions.R is loaded:
source("../_common/functions.R")

# Or define colors manually:
cell_type_color <- scales::hue_pal()(length(unique(combined$cell_type)))
```
**Prevention**: Don't comment out the `source()` call on line 35

### Error: "Cluster indices out of bounds"
**Cause**: Referring to cluster numbers that don't exist  
**Solution**: Check available clusters first:
```r
unique(combined$seurat_clusters)
levels(tmp_ident)
```
**Prevention**: Match cluster indices carefully - remember Seurat starts at 0!

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `min.cutoff` | 'q10' | 'q1'-'q50' | Expression percentile threshold for FeaturePlot |
| `pt.size` | 0.4 | 0.1-2.0 | Point size in UMAP plots |
| `ncol` | 2 | 1-4 | Number of columns in multi-panel plots |

### When to Change Parameters
- **Lower `min.cutoff`**: When markers have low expression (show more cells)
- **Increase `pt.size`**: For small datasets or presentations
- **Adjust `ncol`**: To optimize figure layout for publication
