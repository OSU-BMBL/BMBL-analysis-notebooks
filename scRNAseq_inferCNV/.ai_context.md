# AI Context: scRNAseq InferCNV

## Quick Summary

This workflow infers copy number variations (CNVs) from single-cell RNA-seq data using the inferCNV package. It distinguishes malignant (aneuploid) cells from normal (diploid) cells based on CNV patterns inferred from gene expression across chromosomes. This is particularly useful in cancer studies to identify tumor cells and analyze clonal heterogeneity.

**Analysis Type**: Single-cell RNA-seq CNV inference  
**Primary Tools**: inferCNV, Seurat, SCEVAN  
**Typical Runtime**: 30 minutes - 2 hours (depending on cell count)  
**Memory Requirements**: 16GB minimum, 32GB recommended for datasets >10k cells  
**Platform**: Local only (RStudio) - NOT compatible with OSC

---

## Data Flow

Data transformation pipeline from input to output:

```
Preprocessed Seurat object (combined.qsave)
    ↓ (infercnv_workflow.r)
Extract count matrix and metadata
    ↓
Define reference "normal" cell population
    ↓
Create inferCNV object with gene order file
    ↓
Run CNV inference with denoising and HMM
    ↓
Generate CNV heatmaps and cluster assignments
    ↓
Output: CNV profiles + malignant/normal classification
```

**Input Location**: Working directory with `combined.qsave` and gene order file  
**Output Location**: `./infercnv_1/` directory (configurable)  
**Intermediate Files**: None stored (all in inferCNV object)

## Common Modifications

### 1. Changing Reference Cell Population
**Task**: Define different "normal" reference cells for CNV baseline  
**File**: `infercnv_workflow.r`  
**Line/Variable**: `ref_group_names` in `CreateInferCNVObject()` (around line 44)  
**Change**:
```r
# Current: Uses cluster "11" as reference
ref_group_names=c("11")

# Change to your reference cluster(s):
ref_group_names=c("5")  # Single cluster
ref_group_names=c("5", "8")  # Multiple reference clusters
ref_group_names=c("T_cells", "B_cells")  # Using cell type labels instead of clusters
```
**Impact**: Defines which cells are considered "normal" (diploid baseline). All other cells are compared against this baseline.  
**Testing**: Check CNV heatmap - reference cells should show flat profiles (no CNVs).

### 2. Adjusting Expression Cutoff
**Task**: Optimize for different sequencing platforms  
**File**: `infercnv_workflow.r`  
**Line/Variable**: `cutoff` parameter in `infercnv::run()` (around line 49)  
**Change**:
```r
# For 10x Genomics (low read depth per cell):
infercnv_obj = infercnv::run(infercnv_obj, cutoff = 0.1, ...)

# For Smart-seq2 (higher read depth):
infercnv_obj = infercnv::run(infercnv_obj, cutoff = 1, ...)

# For very sparse data:
infercnv_obj = infercnv::run(infercnv_obj, cutoff = 0.05, ...)
```
**Impact**: Lower cutoff = more genes included, but noisier. Higher cutoff = more stringent, fewer genes but cleaner signal.  
**Testing**: Compare CNV profile smoothness. Too low = noisy; too high = missing real CNVs.

### 3. Enabling/Disabling HMM
**Task**: Control whether to run HMM-based CNV prediction  
**File**: `infercnv_workflow.r`  
**Line/Variable**: `HMM` parameter (around line 54)  
**Change**:
```r
# With HMM (slower, more accurate):
infercnv_obj = infercnv::run(infercnv_obj, HMM = TRUE, ...)

# Without HMM (faster, uses denoising only):
infercnv_obj = infercnv::run(infercnv_obj, HMM = FALSE, ...)
```
**Impact**: HMM adds computational time but provides probabilistic CNV calls. Without HMM, relies on signal thresholding only.  
**Testing**: HMM=TRUE produces more detailed output files with CNV probabilities.

### 4. Changing Number of Threads
**Task**: Speed up computation on multi-core machines  
**File**: `infercnv_workflow.r`  
**Line/Variable**: `num_threads` parameter (around line 55)  
**Change**:
```r
# Default:
infercnv_obj = infercnv::run(infercnv_obj, num_threads = 8, ...)

# Use all available cores:
infercnv_obj = infercnv::run(infercnv_obj, num_threads = parallel::detectCores(), ...)

# Reduce for shared machines:
infercnv_obj = infercnv::run(infercnv_obj, num_threads = 4, ...)
```
**Impact**: More threads = faster runtime, but higher memory usage.  
**Testing**: Monitor system resources with Activity Monitor / htop.

### 5. Changing Output Directory
**Task**: Save results to custom location  
**File**: `infercnv_workflow.r`  **Line/Variable**: `OUT_DIR` variable (around line 33)  
**Change**:
```r
# Default:
OUT_DIR = "./infercnv_1"

# Custom location:
OUT_DIR = "./results/cnv_analysis"
OUT_DIR = "/path/to/output/directory"
```
**Impact**: All output files (heatmaps, matrices, plots) saved to specified directory.  
**Testing**: Verify directory is created and populated after run.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **OSC INCOMPATIBILITY**: inferCNV cannot be installed on OSC cluster
  **Solution**: Must run on local machine or other HPC with full R package support
  **Workaround**: Use SCEVAN (also in script) as alternative - may work on OSC

- **Reference selection is CRITICAL**: Wrong reference = wrong CNV calls
  **Solution**: Use known normal cell types (immune cells, fibroblasts) as reference
  **Best practice**: Multiple independent reference populations if possible

### 🔶 Common Mistakes
- **Using normalized data instead of counts**: inferCNV requires raw counts
  **Why**: Normalization obscures CNV signal
  **Fix**: Code uses `GetAssayData(combined, assay = "RNA", slot = "counts")` - don't change to "data" or "scale.data"

- **Gene order file mismatch**: Using wrong genome build
  **Why**: Gene positions must match your reference genome
  **Fix**: Use `hg38_gencode_v27.txt` for hg38 aligned data, or create custom file for other builds

### 📋 Dependencies Between Steps
- Must have pre-clustered Seurat object BEFORE running
- Must define reference cells BEFORE creating inferCNV object
- Gene order file must match reference genome used in alignment
- RNA assay must be set as default (line 19)

## Common Errors & Solutions

### Error: "Error in library(infercnv) : there is no package called 'infercnv'"
**Cause**: Package not installed  
**Solution**: 
```r
if (!requireNamespace("BiocManager", quietly = TRUE))
    install.packages("BiocManager")
BiocManager::install("infercnv")
```
**Prevention**: Run installation commands at top of script before analysis

### Error: "Gene order file not found"
**Cause**: `hg38_gencode_v27.txt` missing from working directory  
**Solution**: 
```r
# Download gene order file:
# From inferCNV GitHub or create custom
```
**Prevention**: Ensure gene order file is in working directory before running

### Error: "Error in CreateInferCNVObject: reference group names not found"
**Cause**: Specified reference cluster doesn't exist  
**Solution**: 
```r
# Check available clusters:
table(combined$seurat_clusters)

# Use valid cluster name in ref_group_names
```
**Prevention**: Always verify cluster names exist before running

### Error: "cannot allocate vector of size X Gb"
**Cause**: Dataset too large for available memory  
**Solution**: Subsample cells or run on machine with more RAM  
**Prevention**: For datasets >20k cells, consider splitting by batch or cell type

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `cutoff` | 0.1 | 0.05-1.0 | Minimum average expression per gene across cells |
| `cluster_by_groups` | TRUE | TRUE/FALSE | Cluster cells by original groups |
| `denoise` | TRUE | TRUE/FALSE | Apply noise reduction |
| `HMM` | TRUE | TRUE/FALSE | Use HMM for CNV prediction |
| `num_threads` | 8 | 1-32 | Parallel processing threads |

### When to Change Parameters
- **Increase `cutoff`**: For Smart-seq2 data (less sparse than 10x)
- **Disable `denoise`**: If confident in data quality, want raw signal
- **Disable `HMM`**: For faster exploratory analysis
- **Increase `num_threads`**: For large datasets on powerful machines
