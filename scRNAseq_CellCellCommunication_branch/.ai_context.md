# AI Context: scRNAseq Cell-Cell Communication

## Quick Summary

This workflow infers and visualizes cell-cell communication networks from single-cell RNA-seq data using the CellChat R package. It models the probability of communication between cell types by integrating gene expression with prior knowledge of ligand-receptor interactions. The workflow produces publication-ready visualizations including circle plots, chord diagrams, heatmaps, and bubble plots showing signaling pathways between cell populations.

**Analysis Type**: Single-cell RNA-seq cell-cell communication inference  
**Primary Tools**: CellChat, Seurat, NMF, ComplexHeatmap, circlize  
**Typical Runtime**: 15-45 minutes depending on dataset size  
**Memory Requirements**: 16GB minimum, 32GB recommended for datasets >10,000 cells  
**Platform**: Both OSC HPC and Local (RStudio)

---

## Data Flow

Data transformation pipeline from input to output:

```
Normalized gene expression matrix + cell metadata
    ↓ (cellchat.rmd)
Create CellChat object with cell type labels
    ↓
Set ligand-receptor interaction database
    ↓
Identify over-expressed genes and interactions
    ↓
Compute communication probability
    ↓
Aggregate network and infer signaling pathways
    ↓
Generate visualizations (circle plots, heatmaps, chord diagrams)
    ↓
HTML report + figures/ directory with all plots
```

**Input Location**: `./data/` folder (example data included)  
**Output Location**: `cellchat.html` + `figures/` directory  
**Intermediate Files**: None (CellChat object stored in memory)

## Common Modifications

### 1. Changing Cell Group Labels
**Task**: Use different metadata column for cell type grouping  
**File**: `cellchat.rmd`  
**Line/Variable**: `createCellChat()` function, `group.by` parameter (around line 65)  
**Change**:
```r
# Default uses "labels" column:
cellchat <- createCellChat(object = data.input, meta = meta, group.by = "labels")

# Change to your column name:
cellchat <- createCellChat(object = data.input, meta = meta, group.by = "cell_type")
# or
cellchat <- createCellChat(object = data.input, meta = meta, group.by = "seurat_clusters")
```
**Impact**: Changes which cell groupings are used for communication inference.  
**Testing**: Check `unique(cellchat@idents)` shows expected cell types.

### 2. Subsetting to Specific Cell Types
**Task**: Analyze communication only between specific cell populations  
**File**: `cellchat.rmd`  
**Line/Variable**: After data loading, before creating CellChat object (around line 46-58)  
**Change**:
```r
# Subset to specific cell types before analysis
cell_types_of_interest <- c("CD4 T cells", "CD8 T cells", "Macrophages")
cells_to_keep <- meta$labels %in% cell_types_of_interest
data.input <- data.input[, cells_to_keep]
meta <- meta[cells_to_keep, ]
```
**Impact**: Focuses analysis on cell types of interest. Reduces computation time and visual clutter.  
**Testing**: Confirm `length(unique(meta$labels))` equals number of selected cell types.

### 3. Selecting Signaling Database Subset
**Task**: Focus on specific types of signaling (e.g., only secreted signaling)  
**File**: `cellchat.rmd`  
**Line/Variable**: Database subsetting section (around line 88-92)  
**Change**:
```r
# Options:
CellChatDB.use <- subsetDB(CellChatDB, search = "Secreted Signaling")  # Only secreted
CellChatDB.use <- subsetDB(CellChatDB, search = "Cell-Cell Contact")  # Only contact
CellChatDB.use <- subsetDB(CellChatDB, search = "ECM-Receptor")       # Only ECM
CellChatDB.use <- CellChatDB  # Use all (default)
```
**Impact**: Filters which ligand-receptor pairs are considered. Good for focusing on specific biological questions.  
**Testing**: Check `dim(CellChatDB.use$interaction)` shows expected number of interactions.

### 4. Changing Statistical Method for Expression Averaging
**Task**: Adjust sensitivity for detecting weak interactions  
**File**: `cellchat.rmd`  
**Line/Variable**: `computeCommunProb()` call (around line 119)  
**Change**:
```r
# Default (conservative, detects strong interactions):
cellchat <- computeCommunProb(cellchat)

# More sensitive (detects weaker interactions):
cellchat <- computeCommunProb(cellchat, type = "truncatedMean", trim = 0.1)

# Most sensitive:
cellchat <- computeCommunProb(cellchat, type = "truncatedMean", trim = 0.05)
```
**Impact**: Lower trim values detect more interactions but may include noise. Higher values (default) are more conservative.  
**Testing**: Compare number of inferred interactions. Should increase with lower trim values.

### 5. Enabling Cell Population Size Effect
**Task**: Account for cell type abundance in communication strength  
**File**: `cellchat.rmd`  
**Line/Variable**: `computeCommunProb()` call (around line 119)  
**Change**:
```r
# Default (doesn't account for population size):
cellchat <- computeCommunProb(cellchat)

# Account for population size (abundant populations send stronger signals):
cellchat <- computeCommunProb(cellchat, population.size = TRUE)
```
**Impact**: Adjusts communication probability based on cell type proportions. Use when comparing across conditions with different cell type abundances.  
**Testing**: Compare results with/without this setting. Major differences suggest population size is important.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **netClustering parallel bug**: Recent CellChat versions conflict with legacy parallel code
  **Solution**: Add this before visualization functions:
  ```r
  CellChat:::netClustering$do.parallel <- FALSE
  ```
  Or use the trace fix shown in README section 4

- **Species mismatch**: CellChatDB.human vs CellChatDB.mouse
  **Solution**: Always check you're using the correct database for your species
  ```r
  CellChatDB <- CellChatDB.human  # for human data
  CellChatDB <- CellChatDB.mouse  # for mouse data
  ```

### 🔶 Common Mistakes
- **Using count data instead of normalized data**: CellChat requires log-normalized data
  **Why**: Raw counts will give incorrect results
  **Fix**: Ensure data is normalized: `NormalizeData()` in Seurat or equivalent

- **Wrong data orientation**: Genes must be in rows, cells in columns
  **Why**: CellChat expects this orientation
  **Fix**: Check `dim(data.input)` - first number should be ~20k (genes), second should be cell count

### 📋 Dependencies Between Steps
- MUST subset data BEFORE creating CellChat object
- MUST set database BEFORE preprocessing (`subsetData()`)
- MUST run `computeCommunProb()` BEFORE `computeCommunProbPathway()`
- MUST run pathway computation BEFORE `aggregateNet()`

## Common Errors & Solutions

### Error: "unused argument 'mc.cores'"
**Cause**: Legacy parallel code conflict with new CellChat version  
**Solution**: 
```r
CellChat:::netClustering$do.parallel <- FALSE
```
**Prevention**: Always include this line before visualization functions

### Error: "cannot allocate vector of size X Gb"
**Cause**: Memory exhausted (dataset too large)  
**Solution**: Subsample cells or increase system RAM  
**Prevention**: For datasets >20k cells, subset to cell types of interest first

### Error: "No significant interactions"
**Cause**: Too stringent filtering or low expression of signaling genes  
**Solution**: 
```r
# Lower the threshold in filterCommunication
cellchat <- filterCommunication(cellchat, min.cells = 5)  # instead of 10
# Or skip filtering:
# cellchat <- filterCommunication(cellchat, min.cells = 1)
```
**Prevention**: Check expression of known marker genes before running CellChat

### Error: "package 'CellChat' not found"
**Cause**: Installed from CRAN instead of GitHub  
**Solution**: Install from GitHub:
```r
devtools::install_github("sqjin/CellChat")
```
**Prevention**: Follow installation instructions in README exactly

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `min.cells` | 10 | 1-50 | Minimum cells in group to keep communication |
| `type` | "triMean" | "triMean"/"truncatedMean" | Method for averaging expression |
| `trim` | N/A | 0.05-0.25 | Percentage to trim if using truncatedMean |
| `population.size` | FALSE | TRUE/FALSE | Weight by cell group size |

### When to Change Parameters
- **Decrease `min.cells`**: For rare cell populations or small datasets
- **Use `truncatedMean`**: When well-known pathways are missing
- **Increase `trim`**: To be more conservative (fewer, stronger interactions)
- **Enable `population.size`**: When comparing across conditions with different cell type proportions
