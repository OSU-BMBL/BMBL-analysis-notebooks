## Gotchas & Warnings

1. **Arrow files are large**: Each sample creates ~1-5GB Arrow file
2. **Memory intensive**: ArchR keeps data in memory - use `dropCells()` to reduce
3. **Thread control**: Set `threads = 4` to avoid overloading HPC
4. **Genome mismatch**: Must use correct genome or peak calling fails

## Common Errors & Solutions

### Error: "Out of memory"
**Cause**: ATAC data is large  
**Solution**: Reduce `maxCells` parameter or use sparse matrices

### Error: "No peaks found"
**Cause**: Low quality data or wrong genome  
**Solution**: Check fragment files and genome version
