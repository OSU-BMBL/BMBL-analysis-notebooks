# AI Context: scRNAseq Label Transfer

## Quick Summary

This workflow transfers cell type labels from a reference atlas to a query dataset using automated annotation methods. It supports both general label transfer (any reference) and specific Human Lung Cell Atlas (HLCA) annotation using scANVI. This is essential when you have unlabeled single-cell data and want to identify cell types using established reference atlases.

**Analysis Type**: Single-cell RNA-seq cell type annotation  
**Primary Tools**: scANVI, scVI-tools, Scanpy, Seurat  
**Typical Runtime**: 20-60 minutes (depending on dataset size and method)  
**Memory Requirements**: 16GB minimum, 32GB recommended for large datasets (>50k cells)  
**Platform**: Both OSC HPC and Local (Jupyter Notebook / Python)

---

## Data Flow

Data transformation pipeline from input to output:

```
Query dataset (unlabeled, .h5ad/.loom/.rds)
    ↓ (scRNAseq_label_transfer.ipynb)
Reference dataset (pre-annotated)
    ↓
Integration using scANVI or Seurat CCA
    ↓
Label prediction with confidence scores
    ↓
Annotated query dataset with cell type labels
    ↓
Visualization (UMAP with transferred labels)
```

**Input Location**: User-provided paths (query + reference)  
**Output Location**: Same directory as notebook (annotated .h5ad files)  
**Intermediate Files**: 
- Reference model files (downloaded automatically)
- Integrated latent representations
- Nearest neighbors indices

## Common Modifications

### 1. Changing the Reference Atlas
**Task**: Use a different reference dataset instead of HLCA  
**File**: `scRNAseq_label_transfer.ipynb`  
**Line/Variable**: Reference data loading section (around cell 4-7)  
**Change**:
```python
# Instead of HLCA reference:
# url = "https://zenodo.org/record/6337966/files/HLCA_emb_and_metadata.h5ad"
# Use your own reference:
url = "path/to/your/reference.h5ad"
adata_ref = sc.read(url)
```
**Impact**: Changes which cell type labels are transferred. Must ensure reference has consistent annotation schema.  
**Testing**: Check UMAP plots show reasonable mixing of query and reference cells.

### 2. Adjusting Confidence Threshold
**Task**: Filter predictions by confidence score  
**File**: `scRNAseq_label_transfer.ipynb`  
**Line/Variable**: After prediction, filter by uncertainty (around cell 20-25)  
**Change**:
```python
# Add filtering step after prediction:
# Keep only predictions with confidence > 0.7
confidence_threshold = 0.7
adata_query.obs['predicted_labels_filtered'] = adata_query.obs['predicted_labels']
adata_query.obs.loc[adata_query.obs['prediction_confidence'] < confidence_threshold, 
                     'predicted_labels_filtered'] = 'Unknown'
```
**Impact**: Low-confidence predictions marked as "Unknown". Reduces false positives.  
**Testing**: Check distribution of "Unknown" labels - should be <20% of cells typically.

### 3. Using Alternative Annotation Method (HLCA Atlas)
**Task**: Use the HLCA-specific annotation notebook instead of general label transfer  
**File**: `HLCA_atlas_annotation.ipynb` vs `scRNAseq_label_transfer.ipynb`  
**Change**: Use `HLCA_atlas_annotation.ipynb` for lung-specific datasets  
**Key Difference**:
- `HLCA_atlas_annotation.ipynb`: Optimized for HLCA, uses pre-trained scANVI model
- `scRNAseq_label_transfer.ipynb`: General purpose, any reference

**Impact**: HLCA version is faster for lung data and provides more detailed lung-specific annotations.  
**Testing**: Compare annotation quality between methods on your dataset.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **Gene name mismatches**: Reference and query must use same gene naming convention (HGNC symbols recommended)
  **Solution**: Check `adata.var_names` match between datasets. Use `adata.var_names_make_unique()` if needed.

- **Memory limits with large datasets**: scANVI integration can exceed 32GB RAM with >100k cells
  **Solution**: Subsample query to 20k cells for initial testing, or use HPC with 64GB+ RAM

### 🔶 Common Mistakes
- **Wrong species**: Using human reference on mouse data (or vice versa)
  **Why**: Will produce nonsense annotations
  **Fix**: Ensure reference and query are same species, or use ortholog conversion first

- **Batch effects**: Query and reference from different technologies without batch correction
  **Why**: Technical differences mistaken for biological differences
  **Fix**: Use scANVI (handles batch) or Harmony integration before label transfer

### 📋 Dependencies Between Steps
- Must download reference model BEFORE loading query data
- Gene order file must match reference model exactly
- scANVI model must be converted from legacy format if using old checkpoints

## Common Errors & Solutions

### Error: "CUDA out of memory"
**Cause**: GPU memory exhausted during model training  
**Solution**: Set `scvi.settings.batch_size = 128` or use CPU: `scvi.settings.dl_num_workers = 0`  
**Prevention**: Monitor GPU memory with `nvidia-smi`, reduce batch size for large datasets

### Error: "Gene order mismatch"
**Cause**: Query dataset genes don't match reference model gene order  
**Solution**: 
```python
# Ensure gene order matches
adata_query = adata_query[:, adata_ref.var_names].copy()
```
**Prevention**: Always check `adata_query.var_names` against reference before integration

### Error: "scANVI model not found"
**Cause**: Legacy model format not converted  
**Solution**: Run conversion step:
```python
scvi.model.SCANVI.convert_legacy_save("old_model", "new_model")
```
**Prevention**: Use updated model files from HLCA website

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `max_epochs` | 100 | 20-400 | Training epochs for scANVI. Increase for complex datasets |
| `batch_size` | 256 | 128-512 | Larger = faster but more memory |
| `confidence_threshold` | 0.7 | 0.5-0.9 | Minimum confidence to keep prediction |

### When to Change Parameters
- **Increase `max_epochs`**: If integration quality is poor (cells not mixing well)
- **Decrease `batch_size`**: If getting out-of-memory errors
- **Increase `confidence_threshold`**: For publication-quality annotations (fewer false positives)
- **Decrease `confidence_threshold`**: For exploratory analysis (more complete coverage)
