# AI Context: scTCRseq Analysis (Metastasis TCR)

## Quick Summary

This workflow integrates single-cell RNA-seq data with T-cell receptor (TCR) sequencing to analyze clonal expansion and TCR diversity in cancer metastasis. It combines gene expression profiles with immune repertoire data to investigate T-cell dynamics across primary tumors and metastatic sites, enabling analysis of clonal relationships and immune responses in cancer progression.

**Analysis Type**: Single-cell RNA-seq + TCR repertoire analysis  
**Primary Tools**: Scanpy, scirpy, muon, anndata, scipy  
**Typical Runtime**: 30-60 minutes  
**Memory Requirements**: 16GB minimum, 32GB recommended  
**Platform**: Local or OSC HPC (Python/Jupyter)

---

## Data Flow

Data transformation pipeline from input to output:

```
scRNA-seq count matrix + TCR contig annotations
    ↓ (Metastasis_TCR.ipynb)
Load gene expression data (AnnData)
    ↓
Load TCR annotations (filtered_contig_annotations.csv)
    ↓
Merge into MuData object (multi-modal container)
    ↓
TCR clonotype analysis (diversity, expansion)
    ↓
Integration with gene expression (cell type + TCR info)
    ↓
Clonal expansion analysis across conditions
    ↓
Diversity metrics and visualizations
    ↓
Joint GEX + TCR analysis results
```

**Input Location**: TSV/MTX/CSV files (GEX), filtered_contig_annotations.csv (TCR)  
**Output Location**: MuData object with GEX and AIRR (TCR) modalities  
**Intermediate Files**: 
- Separate AnnData objects for GEX
- TCR DataFrames from annotations
- Combined MuData object

### 3. Comparing Conditions/Samples
**Task**: Compare TCR repertoire between primary tumor and metastasis  
**File**: `Metastasis_TCR.ipynb`  
**Line/Variable**: Analysis after MuData creation  
**Change**:
```python

# Add condition labels if not present:

    'primary_tumor': 'Primary',
    'lymph_node': 'Metastasis'
```

**Impact**: Identifies shared vs unique clones between conditions.  
**Testing**: Check that overlap makes biological sense (some shared, some unique).

### 4. Changing Diversity Metrics
**Task**: Calculate different repertoire diversity measures  
**File**: `Metastasis_TCR.ipynb`  
**Change**:
```python

ir.tl.alpha_diversity(mdata, metric='simpson')  # Simpson index
```

**Impact**: Different metrics emphasize different aspects of diversity.  
**Testing**: Compare metrics - they should correlate but capture different properties.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **Barcode matching**: GEX and TCR barcodes must match exactly
  **Solution**: Check barcode formats match between modalities
  **Check**: `set(adata.obs_names).intersection(tcr_df['barcode'])`

- **Multiple chains**: Some cells have multiple productive TCRs
  **Solution**: scirpy handles this, but be aware for interpretation
  **Impact**: Dual TCR cells may appear in multiple clonotypes

### 🔶 Common Mistakes
- **Not filtering for productive contigs**: Non-productive TCRs shouldn't be analyzed
  **Why**: Only productive rearrangements form functional TCRs
  **Fix**: scirpy automatically filters, but verify with:
  ```python
  tcr_df = tcr_df[tcr_df['productive'] == True]
  ```

- **Ignoring chain pairing**: Alpha and beta chains pair specifically
  **Why**: Not all combinations are observed
  **Fix**: Use `ir.pp.ir_dist()` for proper chain analysis

### 📋 Dependencies Between Steps
- Must load GEX data BEFORE adding TCR annotations
- Must have matching barcodes between GEX and TCR
- Clonotype calling must happen before diversity analysis
- Cell type annotation helps interpret TCR results

## Common Errors & Solutions

### Error: "Barcodes don't match between GEX and TCR"
**Cause**: Different barcode formats or filtering  
**Solution**: 
```python
# Check barcode overlap:
common_barcodes = set(adata.obs_names).intersection(set(tcr_df['barcode']))
print(f"Common: {len(common_barcodes)}")

# May need to strip suffixes:
adata.obs_names = [bc.split('-')[0] for bc in adata.obs_names]
```
**Prevention**: Always verify barcode formats match

### Error: "No productive TCR contigs found"
**Cause**: All TCRs filtered as non-productive  
**Solution**: Check productive column:
```python
print(tcr_df['productive'].value_counts())
```
**Prevention**: Ensure input file has productive contigs

### Error: "MuData creation fails"
**Cause**: Dimension mismatch between modalities  
**Solution**: Ensure same observations in both:
```python
# Check alignment:
print(adata.n_obs, tcr_df['barcode'].nunique())
```

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `clonotype threshold` | >1 | 1-10 | Minimum cells for "expanded" clone |
| `diversity metric` | shannon | shannon/simpson/observed | Diversity measure |
| `overlap method` | jaccard | jaccard/cosine/overlap | Repertoire similarity |

### When to Change Parameters
- **Higher threshold**: For strict definition of expansion
- **Lower threshold**: To include more clones in analysis
- **Simpson index**: When concerned about dominant clones
- **Shannon entropy**: For overall diversity measure
