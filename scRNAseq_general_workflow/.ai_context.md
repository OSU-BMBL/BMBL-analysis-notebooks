# AI Context: scRNAseq_general_workflow

## Quick Summary

A comprehensive single-cell RNA-seq analysis workflow using Seurat. Processes raw 10X Genomics data through QC, normalization, clustering, cell type annotation, and differential expression analysis. Designed for OSC HPC cluster execution but works locally too.

**Analysis Type**: Single-cell RNA-seq  
**Primary Tools**: Seurat 5.x, Harmony (optional batch correction), enrichR  
**Typical Runtime**: 30 min - 2 hours (depends on dataset size)  
**Memory Requirements**: 8GB minimum, 16GB recommended  
**Platform**: OSC HPC (Ascend) / Local

---

## Data Flow

```
10X Raw Data (barcodes.tsv.gz, features.tsv.gz, matrix.mtx.gz)
    ↓
Step 1: Package Installation (0_install_packages.R)
    ↓
Step 2: Preprocessing (1_preprocess.rmd)
    - QC filtering (genes, counts, mitochondrial %)
    - Normalization (LogNormalize)
    - Feature selection (highly variable genes)
    - Dimensionality reduction (PCA → UMAP)
    - Clustering (Louvain, resolution 0.8)
    - Optional: Harmony batch correction
    ↓
Step 3: Cell Type Annotation (2_annotate_cell_type.rmd)
    - Marker gene identification (FindAllMarkers)
    - Manual cell type assignment
    ↓
Step 4: Differential Expression (3_deg.rmd)
    - DEG identification (Wilcoxon test)
    - Pathway enrichment (GO, Reactome)
    ↓
Processed Seurat Object + Figures + Results
```

**Input Location**: `data/ctrl_raw_feature_bc_matrix/` and `data/stim_raw_feature_bc_matrix/`  
**Output Locations**:
- `data/processed/` - R objects (.rds)
- `figures/` - Plots (.pdf/.png)
- `results/` - Tables (.csv)

## Common Modifications

### 1. Change Clustering Resolution
**Task**: Adjust number of clusters  
**File**: `1_preprocess.rmd`  
**Variable**: `resolution = 0.8` (in FindClusters function)  
**Change**: Increase to 1.2 for more clusters, decrease to 0.4 for fewer  
**Impact**: Changes cluster assignments, must re-run annotation step  
**Testing**: Visualize UMAP before/after; check cluster sizes

### 2. Modify QC Thresholds
**Task**: Change cell filtering criteria  
**File**: `1_preprocess.rmd`  
**Variables**: 
- `min.features = 200` (minimum genes per cell)
- `max.features = 7000` (maximum genes per cell)
- `max.counts = 30000` (maximum UMIs per cell)
- `percent.mt = 20` (maximum mitochondrial %)
**Impact**: Changes which cells are retained; affects all downstream analysis  
**Testing**: Check QC violin plots; verify cell numbers reasonable

### 3. Enable/Disable Batch Correction
**Task**: Add or remove Harmony batch correction  
**File**: `1_preprocess.rmd`  
**Change**: 
```r
# Add before RunUMAP:
library(harmony)
seurat_object <- RunHarmony(seurat_object, group.by.vars = "batch")

# Then update RunUMAP:
RunUMAP(seurat_object, reduction = "harmony", dims = 1:30)
```
**Impact**: Changes clustering and UMAP; may remove biological variation if over-corrected  
**Testing**: Compare UMAP with/without; check if known cell types still separate

### 4. Change Number of PCs
**Task**: Adjust dimensionality  
**File**: `1_preprocess.rmd`  
**Variable**: `dims = 1:30` (used in FindNeighbors, RunUMAP)  
**Change**: Use 1:50 for complex datasets, 1:20 for simple ones  
**Impact**: Affects clustering granularity and UMAP structure  
**Testing**: Check elbow plot in preprocessing output

### 5. Add Custom Marker Genes
**Task**: Use specific markers for cell type annotation  
**File**: `2_annotate_cell_type.rmd`  
**Approach**: Add to marker gene list, create FeaturePlots  
**Impact**: Cell type assignments  
**Testing**: Cross-reference with known literature markers

## Gotchas & Warnings

### ⚠️ Critical Issues

**Must Run Steps in Order**
- Step 2 (preprocessing) must complete before Step 3 (annotation)
- Step 3 must complete before Step 4 (DEG)
- Each step produces outputs needed by next step

**Seurat Object Continuity**
- The `seurat_object` is the main data carrier
- Changes in Step 2 affect Steps 3 and 4
- Save intermediate objects if experimenting

### 🔶 Common Mistakes

**Running Out of Memory**
- Large datasets (>50k cells) may crash
- **Solution**: Use `Sketch` workflow instead for large data
- **Fix**: Downsample or switch workflows

**Wrong Input Data Structure**
- Expects `data/[sample]_raw_feature_bc_matrix/` folders
- Each needs: barcodes.tsv.gz, features.tsv.gz, matrix.mtx.gz
- **Fix**: Ensure Cell Ranger output structure

**Batch Correction When Not Needed**
- Don't use Harmony if no clear batch effect in PCA
- Over-correction removes real biology
- **Check**: Color UMAP by batch before deciding

**Forgetting to Set Seed**
- Some steps (clustering, UMAP) have randomness
- Results may vary slightly between runs
- **Fix**: Add `set.seed(123)` at start of each script

### 📋 Step Dependencies

1. `1_preprocess.rmd` → creates `seurat_object`
2. `2_annotate_cell_type.rmd` → reads `seurat_object`, adds labels
3. `3_deg.rmd` → reads annotated object, performs DE

If you change preprocessing, you must re-run annotation and DEG steps.

## Common Errors & Solutions

### Error: "cannot allocate vector of size X Gb"
**Cause**: Dataset too large for available RAM  
**Solution**: Use `scRNAseq_Sketch_LargeData` workflow instead, or downsample  
**Prevention**: Check dataset size before starting; use appropriate workflow

### Error: "Feature names cannot have underscores"
**Cause**: Older Cell Ranger versions used underscores in gene names  
**Solution**: Update Cell Ranger or modify gene names  
**Prevention**: Use Cell Ranger v7+ for new data

### Error: "No features to plot" in FeaturePlot
**Cause**: Gene name not in dataset (typo or wrong species)  
**Solution**: Check gene exists: `gene_name %in% rownames(seurat_object)`  
**Prevention**: Verify gene symbols match reference (human vs mouse)

### Error: "Harmony didn't converge"
**Cause**: Batch variable not found or too many batches  
**Solution**: Check `seurat_object$batch` exists; reduce batch complexity  
**Prevention**: Verify metadata before running Harmony

## Parameter Guide

### Key Parameters in Preprocessing

| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `resolution` | 0.8 | 0.1-2.0 | Cluster granularity (higher = more clusters) |
| `dims` | 1:30 | 1:5 to 1:50 | Number of PCs to use |
| `min.features` | 200 | 50-500 | Minimum genes per cell |
| `max.features` | 7000 | 5000-10000 | Maximum genes per cell |
| `percent.mt` | 20 | 5-30 | Maximum mitochondrial percentage |

### When to Change

**Increase `resolution`** (1.0-1.5):
- When you want to identify subpopulations
- For heterogeneous tissues
- More clusters, potentially over-clustering

**Decrease `resolution`** (0.4-0.6):
- When clusters seem too fragmented
- For broad cell type classification
- Fewer, larger clusters

**Increase `dims`** (1:50):
- Complex datasets with many cell types
- When subtle populations are missed
- More computational time

**Adjust QC thresholds**:
- Based on QC plots (violin plots)
- Tissue-specific requirements
- Experiment quality
