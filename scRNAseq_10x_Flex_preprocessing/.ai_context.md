# AI Context: scRNAseq 10x Flex Preprocessing

## Quick Summary

This workflow preprocesses single-cell RNA-seq data from the 10x Genomics Flex platform (fixed RNA profiling). Unlike standard 10x protocols, Flex pools multiple samples during library preparation, requiring demultiplexing after sequencing. The pipeline uses Cell Ranger multi to demultiplex samples, align reads, and generate gene-cell count matrices from raw FASTQ files.

**Analysis Type**: Single-cell RNA-seq preprocessing (10x Flex platform)  
**Primary Tools**: Cell Ranger multi (10x Genomics)  
**Typical Runtime**: 4-10 hours per run (depends on data size)  
**Memory Requirements**: 64GB RAM recommended  
**Platform**: OSC HPC (batch job submission)

---

## Data Flow

Data transformation pipeline from input to output:

```
Raw FASTQ files (multiplexed, pooled samples)
    ↓ (Cell Ranger multi)
Demultiplexing using probe barcodes
    ↓
Alignment to reference genome
    ↓
Gene quantification per cell
    ↓
Filter cells and genes
    ↓
Sample-specific count matrices (barcodes/features/matrix)
```

**Input Location**: Raw FASTQ directory  
**Output Location**: `./<run_id>/outs/` (one per run)  
**Intermediate Files**: Aligned BAM files (temporary)

## Common Modifications

### 1. Creating the Sample CSV Configuration
**Task**: Define sample-to-barcode mappings for demultiplexing  
**File**: `Y12696_GraysonM_Naive-i_V1G_1.csv` (template provided)  
**Structure**:
```csv
[gene-expression]
reference,/path/to/refdata-gex-GRCh38-2020-A
probe-set,/path/to/Chromium_Human_Transcriptome_Probe_Set_v1.0.1_GRCh38-2020-A.csv

[samples]
sample_id,probe_barcode_ids,description
Sample1,BC001,Control_D0
Sample2,BC002,Treatment_D0
Sample3,BC003,Control_D7
```
**Change**: Modify sample_id, probe_barcode_ids, and description for your experiment  
**Impact**: Defines how samples are demultiplexed. Wrong barcodes = wrong sample assignments.  
**Testing**: Verify probe barcodes match what wet lab provided.

### 2. Adjusting Resource Requirements
**Task**: Modify CPU/memory for OSC job  
**File**: `Y12696_GraysonM_Naive-i_V1G_1.sh` (batch script)  
**Line/Variable**: SLURM directives (lines 3-8)  
**Change**:
```bash
# Current:
#SBATCH --time=10:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=8
#SBATCH --mem=64GB

# For larger datasets:
#SBATCH --time=24:00:00
#SBATCH --ntasks=16
#SBATCH --mem=128GB

# For smaller datasets:
#SBATCH --time=4:00:00
#SBATCH --ntasks=4
#SBATCH --mem=32GB
```
**Impact**: Affects runtime and queue priority. More resources = faster but longer queue.  
**Testing**: Monitor first run with `squeue -u $USER` and adjust based on actual usage.

### 3. Changing Cell Ranger Version
**Task**: Use different Cell Ranger version  
**File**: `Y12696_GraysonM_Naive-i_V1G_1.sh`  
**Line/Variable**: CellRanger path (line 92)  
**Change**:
```bash
# Current:
CellRanger=/fs/ess/PCON0022/tools/cellranger-7.1.0/cellranger

# Different version:
CellRanger=/fs/ess/PCON0022/tools/cellranger-8.0.0/cellranger
```
**Impact**: Different versions may have different features and reference genome requirements.  
**Testing**: Check Cell Ranger version compatibility with reference genome and probe set.

### 4. Custom Reference Genome
**Task**: Use non-human reference (mouse, other species)  
**File**: CSV configuration file  
**Line/Variable**: `[gene-expression]` section, `reference` field  
**Change**:
```csv
# Human (default):
reference,/fs/ess/PCON0022/tools/refdata-gex-GRCh38-2020-A

# Mouse:
reference,/fs/ess/PCON0022/tools/refdata-gex-mm10-2020-A

# Custom:
reference,/path/to/your/custom/reference
```
**Impact**: Must match species of your samples. Custom references require pre-building with Cell Ranger mkref.  
**Testing**: Verify reference contains expected gene annotations.

## Gotchas & Warnings

### ⚠️ Critical Issues
- **Probe barcodes must match exactly**: Case-sensitive, no extra spaces
  **Solution**: Copy-paste barcodes from 10x Genomics documentation or lab records
  **Verification**: `cat your_config.csv | grep -i barcode` to check format

- **Sample CSV format is strict**: Must follow exact template structure
  **Solution**: Use provided template, only modify values not structure
  **Common error**: Extra blank lines or missing section headers

### 🔶 Common Mistakes
- **Wrong probe set version**: Must match chemistry version used in lab
  **Why**: Different Flex chemistry versions use different probe sets
  **Fix**: Check 10x Genomics lot number and match to probe set version

- **Insufficient memory allocation**: Default 64GB may not be enough for large datasets
  **Why**: Cell Ranger is memory-intensive
  **Fix**: Increase to 128GB for datasets >10k cells, or check web summary for memory usage

### 📋 Dependencies Between Steps
- Must create CSV config BEFORE running Cell Ranger
- Must have valid reference genome and probe set files
- FASTQ files must be complete (all lanes) before starting

## Common Errors & Solutions

### Error: "Invalid sample CSV format"
**Cause**: CSV doesn't match expected structure  
**Solution**: Use exact template from README, check:
- Section headers in brackets: `[gene-expression]`, `[samples]`
- No extra blank lines
- Correct column names in samples section

### Error: "Out of memory"
**Cause**: Insufficient RAM allocated  
**Solution**: Increase SLURM memory request:
```bash
#SBATCH --mem=128GB
```
**Prevention**: Check 10x Genomics memory calculator for your dataset size

### Error: "Reference genome not found"
**Cause**: Wrong path to reference  
**Solution**: Verify reference path exists:
```bash
ls /fs/ess/PCON0022/tools/refdata-gex-GRCh38-2020-A
```
**Prevention**: Use absolute paths in CSV config

### Error: "Probe barcode not found"
**Cause**: Barcode IDs in CSV don't match probe set file  
**Solution**: Check available barcodes in probe set file, verify spelling

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `--localcores` | 8 | 4-32 | Parallel processing cores |
| `--localmem` | 64 | 32-256 | Memory limit (GB) |
| `--time` | 10:00:00 | 1-48 hours | SLURM time limit |

### When to Change Parameters
- **Increase cores/memory**: For datasets >10k cells or multiple samples
- **Increase time**: If jobs are being killed before completion
- **Decrease resources**: For small test runs (< 1000 cells)
