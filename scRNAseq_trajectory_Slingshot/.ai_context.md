# AI Context: scRNAseq Trajectory Analysis (Slingshot)

## Quick Summary

This workflow infers developmental trajectories from single-cell RNA sequencing data using the Slingshot package. It identifies cell lineages, calculates pseudotime (a measure of progression through a biological process), and visualizes gene expression dynamics along these trajectories. Think of pseudotime as placing cells along a timeline based on their gene expression patterns, even though the actual time isn't known.

**Analysis Type**: Single-cell RNA-seq trajectory inference  
**Primary Tools**: Seurat, Slingshot, tradeSeq, speckle  
**Typical Runtime**: 15-45 minutes depending on dataset size  
**Memory Requirements**: 8GB minimum, 16GB recommended for datasets >10,000 cells  
**Platform**: Both OSC HPC and Local (RStudio)

## Data Flow

Data transformation pipeline from input to output:

```
Pre-analyzed Seurat object (.rds/.qs)
    ↓ (1_trajectory_slingshot.rmd)
Cell type proportion analysis (speckle)
    ↓
Subset to trajectory cell types + re-processing
    ↓
Cluster identification for trajectory start
    ↓
Slingshot trajectory inference
    ↓
Pseudotime calculation
    ↓
Pseudotime distribution plots (density, violin)
    ↓
Gene dynamics visualization (2_pseudotime_gene_dynamics.R)
    ↓
Smoothed expression curves + heatmaps
    ↓
Trajectory results with pseudotime metadata
```

**Input Location**: User-provided Seurat object (any location)  
**Output Location**: `./figures/` directory  
**Intermediate Files**: 
- `figures/00_cell_type_proportion_results.csv` (speckle statistics)
- `figures/XX_UMAP_split_by_group.png` (cluster identification)

## Common Modifications

### 1. Selecting Cell Types for Trajectory
**Task**: Change which cell types to include in the trajectory analysis  
**File**: `1_trajectory_slingshot.rmd`  
**Line/Variable**: `CELL_TYPES_TO_USE` (around line 64)  
**Change**: 
```r
# Current:
CELL_TYPES_TO_USE <- c("Cardiac Fibroblasts", "Myofibroblasts")

# Modify to your cell types:
CELL_TYPES_TO_USE <- c("Progenitor", "Differentiated", "Mature")
```
**Impact**: Defines which cells are included in the trajectory. Must be biologically meaningful (cells should represent a continuous process like differentiation).  
**Testing**: Check `figures/01_UMAP_clusters_and_groups.png` - you should see a continuous progression of cells across the UMAP.

### 2. Setting the Starting Cluster
**Task**: Define where the trajectory begins (the root of the lineage tree)  
**File**: `1_trajectory_slingshot.rmd`  
**Line/Variable**: `start_cluster` (around line 241)  
**Change**:
```r
# After examining figures/02_UMAP_split_by_group.png and marker expression:
start_cluster <- "3"  # Change to your identified starting cluster
```
**Impact**: Determines the direction of pseudotime (0 = start, increasing = progression). Wrong start cluster = reversed biological interpretation.  
**Testing**: Pseudotime should correlate with known marker genes (early markers high at start, late markers increase along trajectory).

### 3. Assigning Experimental Groups
**Task**: Map sample IDs to experimental groups for comparison  
**File**: `1_trajectory_slingshot.rmd`  
**Line/Variable**: Sample lists around lines 87-90  
**Change**:
```r
# Replace these sample ID lists with yours:
group1_samples <- c("your_sample_1", "your_sample_2")
group2_samples <- c("your_sample_3", "your_sample_4")

# Then update the case_when() statement (lines 93-99):
sobj_full[[GROUP_COL]] <- case_when(
  sobj_full[[SAMPLE_COL, drop = TRUE]] %in% group1_samples ~ "Group1",
  sobj_full[[SAMPLE_COL, drop = TRUE]] %in% group2_samples ~ "Group2",
  TRUE ~ "Unknown"
)
```
**Impact**: Groups are used for proportion analysis and pseudotime distribution comparisons.  
**Testing**: Run `table(sobj_full$group)` - should show expected group sizes.

### 4. Selecting Genes for Dynamics Plots
**Task**: Choose which genes to visualize along pseudotime  
**File**: `2_pseudotime_gene_dynamics.R`  
**Line/Variable**: `GENES_TO_PLOT` (line 20)  
**Change**:
```r
# Modify gene list (case-sensitive!):
GENES_TO_PLOT <- c("YourGene1", "YourGene2", "YourGene3")

# Mouse genes: Sentence case (Acta2, Col1a1)
# Human genes: ALL CAPS (ACTA2, COL1A1)
```
**Impact**: Determines which genes appear in smoothed expression plots and heatmaps.  
**Testing**: Check console output - script warns if genes aren't found. Use `rownames(sobj)[grep("pattern", rownames(sobj))]` to find correct gene names.

### 5. Running TradeSeq Differential Analysis
**Task**: Identify genes with different expression patterns between groups along pseudotime  
**File**: `1_trajectory_slingshot.rmd`  
**Section**: Section 6 (lines 285+)  
**Change**: Uncomment and modify tradeSeq analysis code:
```r
# Set contrasts for group comparison
conditions <- factor(sce$group)

# Association test for pseudotime-dependent genes
assoRes <- associationTest(models)
sortedAsso <- assoRes[order(assoRes$pvalue), ]
```
**Impact**: Finds genes with different trajectory dynamics between experimental groups (e.g., treatment delays differentiation).  
**Testing**: Check for significant p-values in output; visualize top hits.

## Gotchas & Warnings

### ⚠️ Critical Issues

- **Case-sensitive gene names**: Mouse genes use sentence case (Acta2, Col1a1), human genes use uppercase (ACTA2, COL1A1). Using wrong case = "gene not found" errors.  
  **Solution**: Always verify gene names with `rownames(sobj)[grep("acta", rownames(sobj), ignore.case=TRUE)]`

- **Wrong start cluster = reversed trajectory**: If pseudotime 0 is actually the END of your biological process, all interpretations are backwards.  
  **Solution**: Verify with known biology - early markers should be high at pseudotime 0, late markers should increase.

- **Over-clustering breaks trajectory**: If you set resolution too high (line 196), cells fragment into too many small clusters, disrupting the continuous trajectory.  
  **Solution**: Use resolution 0.1-0.5 for trajectory; check that clusters form a connected graph in UMAP.

### 🔶 Common Mistakes

- **Including unrelated cell types**: Adding cell types that don't belong to the developmental process (e.g., including immune cells in a fibroblast differentiation trajectory) creates spurious branches.  
  **Why**: Slingshot tries to connect ALL clusters into lineages.  
  **Fix**: Be selective with `CELL_TYPES_TO_USE`; review UMAP before trajectory analysis.

- **Not re-processing after subsetting**: The workflow re-runs normalization and clustering on the subset (lines 187-196). Skipping this uses the original clustering which may not reflect the subset's structure.  
  **Why**: Original UMAP was calculated on all cell types; subset may have different variance structure.  
  **Fix**: Always include the re-processing step (lines 187-196).

- **Assuming one trajectory fits all**: Different experimental groups may have different lineage structures (e.g., KO cells take a different path).  
  **Why**: Slingshot builds one trajectory for all cells combined by default.  
  **Fix**: Consider running separately per group or use tradeSeq's condition tests.

### 📋 Dependencies Between Steps

- Must define `start_cluster` before running Slingshot (lines 241-247)
- Must re-process subset (Section 4) before trajectory inference (Section 5)
- Gene dynamics script (file 2) requires the main script's output or will re-run Slingshot
- Group assignment (Section 2) must be correct for proportion analysis and pseudotime comparisons

## Common Errors & Solutions

### Error: "Gene not found: Xxx"
**Cause**: Gene name case mismatch or gene not in dataset  
**Solution**: 
```r
# Search for gene:
rownames(sobj)[grep("xxx", rownames(sobj), ignore.case=TRUE)]
# Use exact match from output
```
**Prevention**: Always verify gene names before running gene dynamics script.

### Error: "object 'sce' not found"
**Cause**: Trying to run tradeSeq sections without converting to SingleCellExperiment  
**Solution**: Ensure `sce <- as.SingleCellExperiment(sobj_fib)` ran successfully.  
**Prevention**: Run chunks in order; don't skip the conversion step.

### Error: "start cluster not in cluster levels"
**Cause**: `start_cluster` value doesn't match any cluster IDs  
**Solution**: Check cluster IDs with `unique(sobj_fib$seurat_clusters)` and update `start_cluster`.  
**Prevention**: Always examine `figures/02_UMAP_split_by_group.png` before setting start cluster.

### Error: "cannot allocate vector of size X Gb"
**Cause**: Running out of memory (dataset too large for available RAM)  
**Solution**: Run on OSC with more memory, or downsample: `sobj <- subset(sobj, downsample = 3000)`  
**Prevention**: Estimate memory needs: ~1GB per 10,000 cells for trajectory analysis.

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `resolution` | 0.3 | 0.1-1.0 | Clustering resolution. Lower = fewer clusters, smoother trajectory |
| `reducedDim` | UMAP | UMAP/PCA/tSNE | Dimensionality reduction for Slingshot. UMAP recommended |
| `approx_points` | 150 | 50-300 | Number of points for trajectory curves. Higher = smoother |

### When to Change Parameters
- **Increase `resolution`**: When you need finer cell state distinctions
- **Decrease `resolution`**: When trajectory has too many branches or gaps
- **Use PCA instead of UMAP**: If UMAP structure looks distorted; PCA preserves global structure better
