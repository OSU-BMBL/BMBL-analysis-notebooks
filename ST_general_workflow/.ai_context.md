# AI Context: Spatial Transcriptomics General Workflow

## Quick Summary

This workflow analyzes spatial transcriptomics data (specifically 10x Genomics Visium) using Seurat. Unlike single-cell RNA-seq where cells are dissociated, spatial transcriptomics keeps tissue slices intact and measures gene expression at known physical locations. Think of it as a gene expression map overlaid on a tissue image - you can see WHICH cells express WHICH genes and WHERE they are located. This workflow includes standard preprocessing, clustering, integration with scRNA-seq references for cell type annotation, and identification of spatially variable genes.

**Analysis Type**: Spatial transcriptomics (10x Visium)  
**Primary Tools**: Seurat (v5+), SeuratData  
**Typical Runtime**: 30-60 minutes per tissue section  
**Memory Requirements**: 16GB minimum, 32GB recommended for large tissue sections  
**Platform**: Both OSC HPC and Local (RStudio with interactive features)

## Data Flow

Data transformation pipeline from input to output:

```
10x Visium output (spatial folder + filtered_feature_bc_matrix)
    ↓ (ST_general_workflow_tutorial.Rmd)
Load SpatialExperiment/Seurat object
    ↓
Quality Control (nCount_Spatial, mitochondrial %)
    ↓
Normalization (SCTransform recommended)
    ↓
Dimensionality reduction (PCA)
    ↓
Clustering (graph-based)
    ↓
UMAP visualization + Spatial plots
    ↓
Spatially variable feature identification
    ↓
OPTIONAL: Subset anatomical regions
    ↓
OPTIONAL: Integration with scRNA-seq (label transfer)
    ↓
Spatially-resolved cell type maps
    ↓
Annotated spatial data with cell type predictions
```

**Input Location**: 10x Genomics `spatial/` folder + count matrix  
**Output Location**: R environment + exported plots  
**Intermediate Files**: Seurat object with spatial coordinates

**Impact**: Visual clarity of spatial plots.  
**Testing**: Plot should show spots without excessive overlap or gaps.

### 3. Subsetting Anatomical Regions
**Task**: Focus analysis on specific tissue regions (e.g., cortex only)  
**File**: `ST_general_workflow_tutorial.Rmd`  
**Line/Variable**: Subsetting section (lines 140-150)  
**Change**:
```r
# Get tissue coordinates
coords <- GetTissueCoordinates(brain)
coords <- coords[colnames(brain), ]
brain$x <- coords$x
brain$y <- coords$y

# Subset based on spatial coordinates (example: left half)
cortex <- subset(brain, x < 5000)  # Adjust threshold based on your image

# Or subset by cluster identity:
cortex <- subset(brain, idents = c(1, 2, 3, 4))  # Known cortex clusters
```
**Impact**: Focuses analysis on region of interest; removes unrelated tissue.  
**Testing**: `SpatialDimPlot(cortex)` should show only the subset region.

### 4. Integration with scRNA-seq Reference
**Task**: Transfer cell type labels from scRNA-seq to spatial data  
**File**: `Sequencing_Analysis/ST_general_workflow_tutorial.Rmd`  
**Line/Variable**: Integration section (after line 150)  
**Change**:
```r
# Load your annotated scRNA-seq reference
scrna_ref <- readRDS("/path/to/annotated_scrna.rds")

# Find anchors between spatial and scRNA
anchors <- FindTransferAnchors(
  reference = scrna_ref,
  query = brain,
  normalization.method = "SCT",
  features = intersect(rownames(scrna_ref), rownames(brain))
)

# Transfer labels
predictions <- TransferData(
  anchorset = anchors,
  refdata = scrna_ref$cell_type  # Your annotation column
)
brain <- AddMetaData(brain, metadata = predictions)

# Visualize predicted cell types
SpatialFeaturePlot(brain, features = c("predicted.cell_type_score"))
```
**Impact**: Adds cell type annotations to spatial spots based on expression similarity.  
**Testing**: Predictions should match known tissue anatomy (e.g., cortex layers in brain).

### 5. Finding Spatially Variable Features
**Task**: Identify genes with spatial patterns (not just cell type markers)  
**File**: `ST_general_workflow_tutorial.Rmd`  
**Line/Variable**: Section 6 (lines 109-114)  
**Change**:
```r
# Default (markvariogram method):
brain <- FindSpatiallyVariableFeatures(
  brain,
  assay = "SCT",
  features = VariableFeatures(brain)[1:1000],
  selection.method = "markvariogram"
)

# Alternative method (moransi):
brain <- FindSpatiallyVariableFeatures(
  brain,
  assay = "SCT",
  features = VariableFeatures(brain)[1:1000],
  selection.method = "moransi"
)
```
**Impact**: Different methods detect different types of spatial patterns.  
**Testing**: Top features should show clear spatial gradients or clusters on SpatialFeaturePlot.

## Gotchas & Warnings

### ⚠️ Critical Issues

- **Interactive plots need RStudio**: Functions like `SpatialDimPlot(..., interactive = TRUE)` require RStudio viewer pane. Won't work in Jupyter or command line R.  
  **Solution**: Use RStudio Desktop or RStudio Server on OSC. Or set `interactive = FALSE` for static plots.

- **Coordinate system mismatches**: Subsetting by coordinates requires using the correct coordinate names (anterior1_imagerow vs imagerow).  
  **Solution**: Check `colnames(brain@meta.data)` for correct column names; varies by Seurat version.

- **Memory usage with large images**: High-resolution histology images can be 500MB+ per section.  
  **Solution**: Use `image = "lowres"` parameter if available, or process on OSC with more RAM.

### 🔶 Common Mistakes

- **Treating spots as single cells**: Visium spots are 55 micrometers - each spot contains multiple cells (5-10 in dense tissue).  
  **Why**: This is a fundamental limitation of Visium resolution.  
  **Fix**: Use deconvolution methods (see cell type deconv subfolder) to estimate cell type proportions per spot.

- **Using standard scRNA-seq normalization**: Spatial data has different noise characteristics.  
  **Why**: Spatial includes spots with no tissue (background) and variable cell density.  **Fix**: Use SCTransform or spatial-aware normalization; check nCount_Spatial distribution.

- **Ignoring tissue orientation**: Spatial coordinates are arbitrary - rotating the image changes x/y but not biology.  
  **Why**: Different samples may have different orientations.  
  **Fix**: Always visualize with `SpatialDimPlot()` before interpreting coordinate-based subsets.

### 📋 Dependencies Between Steps

- Must run SCTransform (or normalization) before dimensionality reduction
- PCA required before UMAP and clustering
- Clustering should be done before anatomical subsetting (to identify region clusters)
- scRNA-seq reference must be pre-annotated before integration

## Common Errors & Solutions

### Error: "could not find function Load10X_Spatial"
**Cause**: Seurat version < 3.2 or function name changed  
**Solution**: Update Seurat: `Install.packages("Seurat")` or use `Load10X_Spatial` (underscore not dot).  
**Prevention**: Use Seurat v5+ as specified in requirements.

### Error: "no slot of name images"
**Cause**: Seurat object created with older version, structure changed  
**Solution**: Recreate object with current Seurat version or update object: `UpdateSeuratObject(old_obj)`.  
**Prevention**: Always use same Seurat version for analysis.

### Error: "coordinates not found"
**Cause**: Spatial coordinate column names changed in Seurat v5  
**Solution**: Check `GetTissueCoordinates(brain)` and use correct column names (x, y instead of imagerow, imagecol).  
**Prevention**: Follow updated tutorial code for your Seurat version.

### Error: "interactive plots not available"
**Cause**: Not running in RStudio  
**Solution**: Use RStudio Desktop or set `interactive = FALSE`.  
**Prevention**: Check `rstudioapi::isAvailable()` before using interactive features.

## Parameter Guide

### Key Parameters
| Parameter | Default | Range | Description |
|-----------|---------|-------|-------------|
| `pt.size.factor` | 1.0 | 0.1-3.0 | Spot size on spatial plots |
| `alpha` | c(0.1, 1) | 0-1 | Transparency range for expression |
| `dims` | 1:30 | 1:50 | PCA dimensions for clustering |
| `resolution` | 0.8 | 0.4-1.2 | Clustering resolution |

### When to Change Parameters
- **Decrease `pt.size.factor`**: When spots overlap too much (dense tissue)
- **Increase `resolution`**: To identify more subregions
- **Use `dims = 1:50`**: For complex tissues with many cell types
